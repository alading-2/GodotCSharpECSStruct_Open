# Godot ECS 资源配置系统使用手册

> [!NOTE]
> 本文档介绍了位于 `Data/DataNew` 下的新一代配置系统。该系统允许开发者使用 Godot 原生 Inspector 管理游戏数据，同时保持与现有 ECS `Data` 系统的完美兼容。

## 1. 系统简介

本系统旨在解决字典数据 (`Dictionary<string, object>`) **编辑困难**且**类型不安全**的问题。
通过引入 **资源适配器 (Adapter)** 模式，我们实现了：

1.  **可视化编辑**: 像 Unity ScriptableObject 一样在 Inspector 中拖拽、调整数值。
2.  **强类型约束**: 编译期检查字段类型，杜绝拼写错误。
3.  **零侵入性**: 最终输出为 ECS 标准字典，无需修改任何游戏逻辑代码。
4.  **一键迁移**: 提供工具将旧代码数据自动转换为资源文件。

## 2. 目录结构

所有新系统代码位于 `Data/DataNew/`：

```text
Data/DataNew/
├── Core/
│   ├── BindDataKeyAttribute.cs  // [基础] 绑定 C# 属性到 DataKey 字符串
│   └── EntityConfigBase.cs      // [核心] 所有配置的基类，负责自动转换
├── Units/
│   └── EnemyConfig.cs           // [示例] 敌人配置类实现
├── Tools/
│   └── DataMigrationTool.cs     // [工具] 自动生成 .tres 的迁移工具
└── Resources/                   // [数据] 存放生成的 .tres 资源文件
    └── Enemies/
```

## 3. 核心机制

### 3.1 绑定属性 (`BindDataKey`)

我们不再手动写 `dict["BaseHp"] = 100`，而是定义一个 C# 属性并在其上标记特性：

```csharp
// 绑定 DataKey.BaseHp 常量
[BindDataKey(DataKey.BaseHp)]
[Export] public float BaseHp { get; set; } = 100f;
```

### 3.2 自动转换 (`EntityConfigBase`)

基类 `EntityConfigBase` 实现了 `ToDataDictionary()` 方法。
它在运行时（或加载时）利用反射扫描所有带 `[BindDataKey]` 的属性，自动构建出 ECS 需要的字典。
*（注：内部包含缓存机制，反射开销仅发生在首次加载。）*

## 4. 使用指南

### 4.1 定义新的配置类

新建一个继承自 `EntityConfigBase` 的类，并标记 `[GlobalClass]`：

```csharp
using Godot;
using Brotato.Data.Config.Core;

[GlobalClass]
public partial class WeaponConfig : EntityConfigBase
{
    [ExportGroup("属性")]
    [BindDataKey(DataKey.Damage)]
    [Export] public float Damage { get; set; }
    
    [BindDataKey(DataKey.IconPath)]
    [Export] public Texture2D Icon { get; set; }
    
    // 处理特殊转换逻辑
    protected override void OnConvertToData(Dictionary<string, object> data)
    {
        if (Icon != null) data[DataKey.IconPath] = Icon.ResourcePath;
    }
}
```

### 4.2 创建数据资源

1.  在 Godot **FileSystem** 面板右键 -> **Create New...** -> **Resource**。
2.  搜索你的类名（如 `EnemyConfig`）。
3.  保存为 `.tres` 文件（如 `Orc.tres`）。
4.  在 **Inspector** 面板中随意修改数值。

### 4.3 在游戏中使用

在生成实体的地方（如 `EntityManager` 或 `Spawner`），加载资源并转换：

```csharp
// 1. 加载资源 (支持热重载!)
var config = GD.Load<EnemyConfig>("res://Data/DataNew/Resources/Enemies/Orc.tres");

// 2. 转换为 ECS 字典
var dataDict = config.ToDataDictionary();

// 3. 传入 ECS (与旧系统完全兼容)
EntityManager.Spawn<EnemyEntity>(new EntitySpawnConfig {
    Config = dataDict,
    Position = ...
});
```

## 5. 迁移工具使用

如果你有大量的旧代码数据 (`EnemyData.cs`)，请使用迁移工具自动生成资源：

1.  **编译项目**。
2.  新建一个空的 Node，挂载 `Data/DataNew/Tools/DataMigrationTool.cs`。
3.  点击 Node，在 Inspector 中勾选 **Run Migration**。
4.  检查 `Data/DataNew/Resources/Enemies/` 目录，所有敌人数据已自动生成为 `.tres` 文件。

## 6. 常见问题 (Q&A)

*   **Q: 只有部分属性想导出怎么办？**
    *   A: 只有标记了 `[BindDataKey]` 的属性会被自动转换。其他 `[Export]` 属性仅作为中间变量存在。
*   **Q: 如何处理复杂的嵌套数据（如 SpawnRule）？**
    *   A: 重写 `OnConvertToData` 方法，在该方法中手动将复杂对象处理好塞入字典。
*   **Q: 性能如何？**
    *   A: 反射结果会被静态缓存。同一个配置类实例化 1000 次，反射只会在第 1 次执行，后续都是毫秒级的直接内存访问。

# 碰撞系统说明

## 一、系统定位

当前项目的碰撞系统采用：

- **模板资源层**：`Src/ECS/Component/Presets/Collision/`
- **统一事件桥接层**：`CollisionComponent`
- **业务消费层**：`ContactDamageComponent`、移动系统、拾取/命中类组件

核心目标不是把所有碰撞都塞进一个巨型组件，而是把碰撞拆成三件事：

- **我身上有哪些碰撞节点**
- **哪个碰撞节点触发了事件**
- **碰撞之后业务上要做什么**

## 二、核心结论

### 2.1 统一碰撞桥接入口已经收敛

当前架构不再保留独立的旧式传感器桥接组件。

现在统一由 `CollisionComponent` 负责：

- 扫描 `VisualRoot` 注入的碰撞模板
- 扫描实体根节点下 `Collision` 容器中的传感器模板
- 把 Godot 原生碰撞信号转发为 `Entity.Events`

这意味着当前碰撞架构的统一入口是：

- `Src/ECS/Component/Collision/CollisionComponent/CollisionComponent.cs`

### 2.2 接触伤害不再耦合碰撞采样

`ContactDamageComponent` 不再自己探测碰撞，只消费：

- `CollisionEntered`
- `CollisionExited`

并按 `CollisionType.Hurtbox` 过滤自己关心的受击感应区。

### 2.3 视觉碰撞模板与实体传感器模板统一进 `Presets/Collision`

所有碰撞模板统一放在：

`Src/ECS/Component/Presets/Collision/`

其中：

- `Unit/`：玩家/敌人视觉体碰撞模板
- `Effect/`：特效或飞行体视觉碰撞模板
- `Sensor/`：受击区/拾取区等传感器模板

## 三、系统分层

## 3.1 模板资源层

目录：`Src/ECS/Component/Presets/Collision/`

职责：

- 提供可实例化的 `.tscn` 模板
- 统一 layer/mask 语义来源
- 作为 `SpriteFramesGenerator` 与 `ResourceGenerator` 的输入目录

关键文件：

- `CollisionType.cs`
- `CollisionTypeQuery.cs`
- `CollisionTypeRegistry.cs`
- `Unit/*.tscn`
- `Effect/*.tscn`
- `Sensor/*.tscn`

## 3.2 事件桥接层

组件：`CollisionComponent`

职责：

- 绑定 `Area2D` 的进入/离开信号
- 识别是哪个碰撞节点触发了事件
- 向 `Entity.Events` 发标准化碰撞事件

输出事件：

- `GameEventType.Collision.CollisionEntered`
- `GameEventType.Collision.CollisionExited`

## 3.3 业务消费层

### 接触伤害

组件：`ContactDamageComponent`

职责：

- 监听本实体的碰撞事件
- 只处理 `CollisionType.Hurtbox`
- 敌对实体进入后立即伤害一次
- 使用 `TimerManager` 维持持续接触伤害

### 移动命中销毁

组件：`EntityMovementComponent`

职责：

- 在非默认运动模式下监听碰撞
- 发布 `MovementCollision`
- `DestroyOnCollision = true` 时完成移动并销毁实体

### 其它碰撞业务

例如：

- 拾取
- 命中特效
- 技能区域效果
- 触碰爆炸

都应建立在统一碰撞事件之上，而不是重新直连 Godot 原生信号。

## 四、节点结构约定

## 4.1 视觉碰撞节点

对于由 `SpriteFramesGenerator` 生成的视觉场景：

```text
VisualRoot
  └─ CollisionShape2D              ← 碰撞模板实例，名称固定
      └─ CollisionShape2D          ← 生成器首次注入的默认形状
```

说明：

- 外层节点可能是 `CharacterBody2D` 或 `Area2D`
- 外层节点提供 layer/mask 语义
- 内层 `CollisionShape2D` 提供实际碰撞形状
- 智能更新时保留手工调整的形状与 transform

## 4.2 实体传感器节点

对于实体自己的受击区、拾取区：

```text
Entity Root
  └─ Collision
      ├─ PlayerHurtboxSensor
      │   └─ CollisionShape2D
      ├─ EnemyHurtboxSensor
      │   └─ CollisionShape2D
      └─ PlayerPickupSensor
          └─ CollisionShape2D
```

说明：

- `Collision` 容器节点名固定
- 子节点来自 `Presets/Collision/Sensor/`
- `CollisionComponent` 会统一扫描并接入事件总线

## 五、CollisionType 语义

碰撞系统不依赖节点名做业务判断，而是通过节点自身：

- `collision_layer`
- `collision_mask`

反查 `CollisionType`。

### 5.1 单场景类型

例如：

- `EffectCollision`
- `EnemyCollision`
- `EnemyHurtboxSensor`
- `PlayerCollision`
- `PlayerHurtboxSensor`
- `PlayerPickupSensor`

### 5.2 组合语义类型

例如：

- `VisualBody`
- `Hurtbox`
- `Pickup`
- `AnySensor`

### 5.3 事件中的 `CollisionType` 表示什么

事件里的 `CollisionType` 表示：

> **本方被触发的那个碰撞节点的语义类型**

不是目标对象的类型。

这点非常重要，因为业务层常常要回答的是：

- 是我的受击区被碰到了？
- 是我的拾取区被碰到了？
- 还是我的视觉碰撞体碰到了？

## 六、自动生成链路

## 6.1 `SpriteFramesGenerator`

配置文件：

`addons/SpriteFramesGenerator/sprite_frames_config.gd`

其中 `RULES[*].collision_scene_path` 直接指向碰撞模板场景。

当前典型配置：

- Player → `Unit/PlayerCollision.tscn`
- Enemy → `Unit/EnemyCollision.tscn`
- Effect → `Effect/EffectCollision.tscn`

作用：

- 把碰撞模板注入生成的 `AnimatedSprite2D` 场景
- 自动补一个默认 `CollisionShape2D`
- 保留手调形状，不在重生成时覆盖

## 6.2 `ResourceGenerator`

工具：`Tools/ResourceGenerator/ResourceGenerator.cs`

作用：

- 扫描 `Src/ECS/Component/Presets/Collision/` 下所有 `.tscn`
- 解析根节点名、`collision_layer`、`collision_mask`
- 自动生成 `CollisionTypeRegistry.cs`

这保证了运行时可以稳定完成：

- `CollisionType -> (Layer, Mask)`
- `(Layer, Mask) -> CollisionType`

双向查询。

## 七、接触伤害链路

```text
实体挂有 CollisionComponent + ContactDamageComponent
  ↓
本实体的 Hurtbox Sensor 触发 CollisionEntered
  ↓
CollisionComponent 发出 Entity.Events.CollisionEntered
  ↓
ContactDamageComponent 过滤 CollisionType.Hurtbox
  ↓
判断 Target 是否敌对实体
  ↓
立即造成一次伤害
  ↓
按 AttackInterval 启动循环 Timer
  ↓
持续接触期间重复结算伤害
  ↓
CollisionExited 时移除对应 Timer
```

关键点：

- 是“该实体碰到敌人就受伤害”，不是“它主动去给敌人上伤害”
- 伤害结算统一走 `DamageService.Instance.Process()`
- `ContactDamageComponent` 只关心敌对关系和节流，不关心原生碰撞查询

## 八、与移动系统的关系

移动系统并不拥有碰撞系统，但依赖碰撞系统提供统一事件。

### 8.1 `Area2D` 路径

`EntityMovementComponent` 订阅 `CollisionEntered`。

适用于：

- 特效
- 飞行体
- 以 `Area2D` 为主体的运动实体

### 8.2 `CharacterBody2D` 路径

`EntityMovementComponent` 在 `MoveAndSlide()` 后通过 `GetSlideCollisionCount()` 检测碰撞。

适用于：

- 玩家
- 敌人
- 其它 `CharacterBody2D` 运动实体

### 8.3 统一语义

不论哪条路径，只要触发的是非默认运动模式的碰撞：

- 先发 `MovementCollision`
- 再根据 `DestroyOnCollision` 决定是否完成并销毁

## 九、推荐实践

## 9.1 新增碰撞语义时

优先走模板体系：

1. 在 `Presets/Collision/` 新增 `.tscn`
2. 补 `CollisionType.cs`
3. 跑 `ResourceGenerator`
4. 在业务组件按 `CollisionType` 消费

## 9.2 不要在业务组件里重复直连原生信号

以下逻辑应优先监听 `Entity.Events`：

- 接触伤害
- 拾取
- 技能命中
- 运动命中即销毁

## 9.3 不要把碰撞形状重新硬编码回多个实体场景

优先维护：

- `Presets/Collision/` 模板
- `sprite_frames_config.gd` 规则
- 生成后的视觉场景局部手调形状

# Godot 碰撞层级说明（本项目）

## 一、Layer 和 Mask 是什么？

> **Layer = 我是谁，Mask = 我关心谁**

- **`collision_layer`**：当前节点挂在哪些身份层上
- **`collision_mask`**：当前节点会去检测哪些层

只要 A 的 `mask` 命中 B 的 `layer`，或者 B 的 `mask` 命中 A 的 `layer`，碰撞就可能成立。

## 二、本项目的 8 个层定义

```text
Layer 1 (值=1)    → Terrain          地形/障碍物
Layer 2 (值=2)    → Player           玩家物理体
Layer 3 (值=4)    → Enemy            敌人物理体
Layer 4 (值=8)    → PlayerHurtbox    玩家受击感应区
Layer 5 (值=16)   → PlayerPickup     玩家拾取感应区
Layer 6 (值=32)   → Projectile       子弹/投射物
Layer 7 (值=64)   → EnemyHurtbox     敌人受击感应区
Layer 8 (值=128)  → WeaponHitbox     武器/技能命中区
```

所有常量统一定义在：

`Data/DataKey/Base/CollisionLayers.cs`

代码中禁止直接写魔法数字。

## 三、当前碰撞架构

### 3.1 统一入口：`CollisionComponent`

当前项目已经统一由 `CollisionComponent` 负责碰撞采样与事件桥接。

它会扫描两类碰撞来源：

- `VisualRoot/CollisionShape2D`：由 `SpriteFramesGenerator` 注入的视觉碰撞模板
- `Entity/Collision`：实体根节点下的传感器模板容器

然后统一向 `Entity.Events` 发出：

- `CollisionEntered`
- `CollisionExited`

### 3.2 碰撞模板目录

所有碰撞模板统一存放于：

`Src/ECS/Component/Presets/Collision/`

```text
Src/ECS/Component/Presets/Collision/
  Unit/
    EnemyCollision.tscn
    PlayerCollision.tscn
  Effect/
    EffectCollision.tscn
  Sensor/
    EnemyHurtboxSensor.tscn
    PlayerHurtboxSensor.tscn
    PlayerPickupSensor.tscn
```

语义分工：

- `Unit/`：玩家/敌人视觉体碰撞模板
- `Effect/`：特效或飞行体碰撞模板
- `Sensor/`：受击区、拾取区等传感器模板

### 3.3 自动生成链路

#### SpriteFramesGenerator

`addons/SpriteFramesGenerator/sprite_frames_config.gd` 中的 `RULES[*].collision_scene_path` 会引用这些模板。

生成后的标准结构：

```text
AnimatedSprite2D
  └─ CollisionShape2D          ← 碰撞模板实例，名称固定
      └─ CollisionShape2D      ← 生成器首次注入的默认形状
```

#### ResourceGenerator

`Tools/ResourceGenerator/ResourceGenerator.cs` 会扫描 `Presets/Collision` 下所有 `.tscn`，自动生成：

- `CollisionTypeRegistry.cs`

用于提供：

- `CollisionType -> (Layer, Mask)`
- `(Layer, Mask) -> CollisionType`

双向查询。

### 3.4 CollisionType 的真实语义

当前 `CollisionType` 由节点自身的 `collision_layer / collision_mask` 反查得到。

事件中的 `CollisionType` 表示：

> **我身上的哪个碰撞节点被触发了**

不是目标对象的类型。

例如：

- `PlayerHurtboxSensor`
- `EnemyHurtboxSensor`
- `PlayerPickupSensor`
- `EffectCollision`

## 四、实体节点结构约定

### 4.1 角色实体

```text
Player / Enemy (CharacterBody2D)
  ├─ Component
  │   ├─ CollisionComponent
  │   └─ ContactDamageComponent（可选）
  ├─ Collision
  │   ├─ PlayerHurtboxSensor / EnemyHurtboxSensor / PlayerPickupSensor
  │   │   └─ CollisionShape2D
  ├─ CollisionShape2D          ← 根物理体碰撞形状
  └─ VisualRoot
      └─ CollisionShape2D      ← 生成器注入的视觉碰撞模板实例
          └─ CollisionShape2D  ← 模板实例下的实际形状
```

### 4.2 特效 / 飞行体实体

```text
EffectEntity / BulletEntity
  ├─ Component
  │   ├─ CollisionComponent
  │   └─ EntityMovementComponent
  └─ VisualRoot
      └─ CollisionShape2D
          └─ CollisionShape2D
```

## 五、各对象的 Layer / Mask 设置一览

| 对象 | 节点/组件 | collision_layer | collision_mask | 配置来源 |
| ---- | ---- | -------------- | -------------- | ---- |
| **PlayerEntity 根物理体** | `CharacterBody2D` | `2` (Player) | `1` (Terrain) | 实体场景 |
| **EnemyEntity 根物理体** | `CharacterBody2D` | `4` (Enemy) | `1+4` (Terrain+Enemy) | 碰撞模板或实体场景 |
| **玩家受击区** | `PlayerHurtboxSensor` | `8` (PlayerHurtbox) | `4` (Enemy) | `Presets/Collision/Sensor` |
| **敌人受击区** | `EnemyHurtboxSensor` | `64` (EnemyHurtbox) | `128` (WeaponHitbox) | `Presets/Collision/Sensor` |
| **玩家拾取区** | `PlayerPickupSensor` | `16` (PlayerPickup) | `0` | `Presets/Collision/Sensor` |
| **特效碰撞体** | `EffectCollision` | `0` | `0` | `Presets/Collision/Effect`，业务再补 mask |
| **子弹/投射物** | `Area2D`/`CharacterBody2D` | `32` (Projectile) | 视业务而定 | 业务实体或模板 |
| **武器命中区** | `Area2D` | `128` (WeaponHitbox) | `64` (EnemyHurtbox) | 业务组件 |

## 六、核心碰撞链路

### 链路 1：接触伤害

```text
实体挂有 Hurtbox Sensor + CollisionComponent + ContactDamageComponent
  ↓
Hurtbox Sensor 触发原生碰撞
  ↓
CollisionComponent 转发为 CollisionEntered
  ↓
ContactDamageComponent 过滤 CollisionType.Hurtbox
  ↓
若 Target 为敌对实体，则自己受到接触伤害
```

关键点：

- `CollisionComponent` 只负责桥接
- `ContactDamageComponent` 只负责伤害节流与结算
- 伤害统一走 `DamageService.Instance.Process()`

### 链路 2：玩家拾取掉落物

```text
玩家的 PlayerPickupSensor 触发碰撞
  ↓
CollisionComponent 转发为 CollisionEntered
  ↓
拾取业务组件按 CollisionType.Pickup 过滤
  ↓
触发磁吸 / 拾取逻辑
```

### 链路 3：飞行体命中即销毁

```text
Area2D 型飞行体运动中发生碰撞
  ↓
CollisionComponent 发出 CollisionEntered
  ↓
EntityMovementComponent 在非默认运动模式下处理
  ↓
发出 MovementCollision
  ↓
若 DestroyOnCollision = true，则完成并销毁实体
```

### 链路 4：CharacterBody2D 物理碰撞

```text
CharacterBody2D 执行 MoveAndSlide()
  ↓
EntityMovementComponent 读取 GetSlideCollisionCount()
  ↓
发出 MovementCollision
```

这条路径不依赖 `Area2D` 信号桥接，而是走物理移动检测。

## 七、碰撞关闭时机

| 时机 | 操作 | 原因 |
| ---- | ---- | ---- |
| Entity 死亡 | `Monitoring = false` 或关闭形状 | 防止死亡后继续参与感应 |
| 对象池归还 | `ProcessMode = Disabled` + 关闭碰撞形状 | 彻底退出活跃逻辑与物理参与 |
| 无敌帧 | `Monitoring = false` | 暂停主动感应 |
| 子弹命中后 | `Monitoring = false` 或销毁实体 | 防止重复命中 |

需要区分：

- `Monitoring = false`：自己不再主动检测别人
- `Monitorable = false`：别人不再能检测到自己
- `CollisionShape2D.disabled = true`：彻底关闭该碰撞形状
- `ProcessMode = Disabled`：关闭节点逻辑回调，但**不能单独替代物理停用**

## 八、为什么 Enemy 仍然保持 `CharacterBody2D`

| 需求 | CharacterBody2D | Area2D |
| ---- | --------------- | ------ |
| 被地形阻挡 | ✅ | ❌ |
| 敌人之间互相推开 | ✅ | ❌ |
| 常驻物理移动 | ✅ | ❌ |
| 受击区 / 命中区感应 | 配合 Area2D 模板完成 | ✅ |

所以项目中采用：

- **物理移动** 用 `CharacterBody2D`
- **受击 / 拾取 / 命中感应** 用 `Area2D`

二者职责分离。

## 九、设计总结

- 碰撞模板统一放在 `Src/ECS/Component/Presets/Collision/`
- 统一由 `CollisionComponent` 把碰撞节点接入 `Entity.Events`
- `ContactDamageComponent` 只做接触伤害
- `EntityMovementComponent` 只做移动碰撞完成与销毁
- `SpriteFramesGenerator` 负责模板注入
- `ResourceGenerator` 负责生成 `CollisionTypeRegistry.cs`
- 代码统一使用 `CollisionLayers` 与 `CollisionType`，禁止魔法数字和字符串语义

## 十、快速设置参考

```csharp
// 推荐：使用常量而不是直接写数字
CollisionLayer = CollisionLayers.PlayerHurtbox;
CollisionMask = CollisionLayers.Enemy;

// 暂停主动检测
Monitoring = false;

// 不允许被别人感知
Monitorable = false;
```

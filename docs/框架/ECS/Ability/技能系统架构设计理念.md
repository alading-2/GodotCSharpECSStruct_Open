# 技能系统架构设计理念

**文档类型**：架构设计  
**目标受众**：架构师、新成员、AI 助手  
**最后更新**：2026-01-16

---

## 核心理念

技能系统是游戏战斗系统的重要组成部分，负责管理玩家和单位的所有技能（主动技能、被动技能、武器技能等）。

**核心原则**：

- **Ability = Entity**：每个技能都是独立的实体，实现 `IEntity` 接口
- **配置驱动**：技能属性通过 `Data` 容器管理，支持动态修改
- **效果解耦**：技能效果（伤害、Buff、召唤等）通过 `AbilityEffect` 执行器实现
- **触发条件化**：支持多种触发方式（手动、自动、条件触发）
- **统一管理**：通过 `AbilitySystem` 管理技能的生命周期和执行

## 主动 vs 被动技能的本质区别

> [!IMPORTANT]
> **本质区别不在于"能做什么"，而在于"如何触发"和"资源管理"**

### 深度思考

传统理解中，人们常认为：
- 主动技能 = 复杂效果（伤害、召唤）
- 被动技能 = 简单效果（属性加成）

**但这是错误的！**

实际上，被动技能同样可以：
- ✅ 造成范围伤害（如燃烧光环，每 0.5 秒对周围敌人造成伤害）
- ✅ 有内部冷却（如反伤，0.5 秒内最多触发一次）
- ✅ 召唤单位（如击杀敌人时 10% 概率召唤骷髅）
- ✅ 施加 Buff（如受击时获得护盾）

### 真正的区别

| 特性 | 主动技能 | 被动技能 |
|:---|:---|:---|
| **触发方式** | 需要玩家输入 | 自动触发 |
| **充能系统** | ✅ 通常有充能 | ❌ 无充能概念 |
| **冷却系统** | ✅ 有冷却 | ✅ 可以有内部冷却 |
| **效果类型** | 任意（伤害/召唤/Buff/位移） | 任意（伤害/召唤/Buff/属性） |
| **资源消耗** | ✅ 可能消耗资源 | ❌ 通常无消耗 |

### 分类定义

```
┌─────────────────────────────────────────────────────────────────┐
│                         技能类型分类                              │
├─────────────────────────────────────────────────────────────────┤
│  主动技能 (Active)                                               │
│  ├─ 触发方式: 需要玩家输入 (按键)                                  │
│  ├─ 充能: ✅ 可以有多次充能 (如冲刺 2 次)                          │
│  ├─ 冷却: ✅ 有                                                  │
│  ├─ 消耗: ✅ 可能消耗资源 (能量/弹药)                              │
│  └─ 示例: 冲刺、手雷、召唤炮台、治疗术                             │
├─────────────────────────────────────────────────────────────────┤
│  被动技能 (Passive)                                              │
│  ├─ 触发方式: 自动触发 (事件/周期/永久)                            │
│  ├─ 充能: ❌ 无充能概念                                          │
│  ├─ 冷却: ✅ 可以有内部冷却 (防止过于频繁)                         │
│  ├─ 消耗: ❌ 通常无消耗                                          │
│  └─ 示例: 生命偷取、反伤、燃烧光环、击杀回血                        │
├─────────────────────────────────────────────────────────────────┤
│  武器技能 (Weapon)                                               │
│  ├─ 触发方式: 自动触发 (与自动攻击绑定)                            │
│  ├─ 充能: ❌ 无充能概念                                          │
│  ├─ 冷却: ✅ 受攻击速度影响                                       │
│  ├─ 消耗: ✅ 可能消耗弹药                                         │
│  └─ 示例: 近战攻击、远程射击、霰弹枪                               │
└─────────────────────────────────────────────────────────────────┘
```

## 被动技能的触发模式

被动技能虽然不需要玩家输入，但有多种触发方式：

| 触发模式 | 描述 | 示例 |
|:---|:---|:---|
| **永久生效** | 始终生效，无触发概念 | 攻击力 +10%、暴击率 +5% |
| **事件触发** | 监听特定事件，条件满足时触发 | 受击反伤、击杀回血、闪避时加速 |
| **周期触发** | 固定时间间隔自动触发 | 燃烧光环 (每 0.5 秒伤害)、回血光环 (每 1 秒恢复) |

### 事件触发的被动技能示例

```
被动技能: 反伤 (Thorns)
├─ 触发方式: 事件触发 (监听 Unit.Damaged)
├─ 内部冷却: 0.5 秒
├─ 效果: 对攻击者造成 10 点伤害
└─ 实现:
   TriggerComponent:
     - TriggerMode = OnEvent
     - TriggerEvent = GameEventType.Unit.Damaged
   CooldownComponent:
     - AbilityCooldown = 0.5f (内部冷却)
   Effect:
     - DamageEffect → 目标 = 攻击者
```

### 周期触发的被动技能示例

```
被动技能: 燃烧光环 (Burn Aura)
├─ 触发方式: 周期触发 (每 0.5 秒)
├─ 效果: 对 100 范围内敌人造成 5 点伤害
└─ 实现:
   TriggerComponent:
     - TriggerMode = Periodic
     - TriggerInterval = 0.5f
   Effect:
     - DamageEffect
     - TargetType = AreaOfEffect
     - Range = 100f
```

### 组件使用矩阵

| 组件 | 主动技能 | 被动技能 | 武器技能 | 说明 |
|:---|:---:|:---:|:---:|:---|
| CooldownComponent | ✅ | ✅ | ✅ | 所有技能都可能需要冷却 |
| ChargeComponent | ✅ | ❌ | ❌ | 仅主动技能有充能概念 |
| CostComponent | ✅ | ❌ | ⚠️ | 主动必有，武器可能消耗弹药 |
| TriggerComponent | ✅ | ✅ | ✅ | 核心区分组件，决定触发方式 |
| TargetingComponent | ⚠️ | ⚠️ | ✅ | 需要选择目标时使用 |

## 核心组件设计

### 1. TriggerComponent (触发组件) - 核心区分组件

**职责**：决定技能如何触发，是区分主动/被动的关键组件

**触发模式**：
```csharp
public enum AbilityTriggerMode
{
    // 主动技能触发
    Manual,         // 手动触发 - 需要玩家按键输入
    
    // 被动技能触发
    OnEvent,        // 事件触发 - 监听特定事件 (如受击、击杀)
    Periodic,       // 周期触发 - 固定时间间隔 (如光环)
    Permanent,      // 永久生效 - 无触发概念 (如属性加成)
    
    // 武器技能触发
    Auto            // 自动触发 - 满足条件自动释放 (如自动攻击)
}
```

**Data 依赖**：
- `DataKey.AbilityTriggerMode`: 触发模式
- `DataKey.AbilityTriggerEvent`: 监听的事件类型 (OnEvent 模式)
- `DataKey.AbilityTriggerInterval`: 触发间隔 (Periodic 模式)
- `DataKey.AbilityTriggerChance`: 触发概率 (可选)

### 2. CooldownComponent (冷却组件)

**职责**：管理技能的冷却时间，主动和被动技能都可能使用

**对于主动技能**：
- 使用后进入冷却
- 冷却期间不可再次使用

**对于被动技能 (内部冷却)**：
- 触发后进入冷却
- 防止效果过于频繁 (如反伤 0.5 秒内最多触发一次)

**Data 依赖**：
- `DataKey.AbilityCooldown`: 基础冷却时间
- `DataKey.CooldownReduction`: 冷却缩减百分比 (支持修改器)

### 3. ChargeComponent (充能组件)

**职责**：管理技能的充能次数，**仅主动技能使用**

**设计原因**：
- 充能是"可积累的使用次数"
- 被动技能不需要"使用"的概念，自然也不需要充能

**Data 依赖**：
- `DataKey.AbilityMaxCharges`: 最大充能次数
- `DataKey.AbilityCurrentCharges`: 当前充能次数
- `DataKey.AbilityChargeTime`: 单次充能恢复时间

### 4. CostComponent (消耗组件)

**职责**：管理技能的资源消耗，**主要用于主动技能**

**Data 依赖**：
- `DataKey.AbilityCostType`: 消耗类型 (Mana, Energy, Ammo, Health)
- `DataKey.AbilityCostAmount`: 消耗数量

### 5. TargetingComponent (目标选择组件)

**职责**：管理技能的目标选择逻辑

**目标类型**：
```csharp
public enum AbilityTargetType
{
    Self,           // 自身
    SingleEnemy,    // 单个敌人 (最近/随机)
    AllEnemies,     // 所有敌人
    AreaOfEffect,   // 区域范围
    Projectile,     // 投射物路径
    Cursor,         // 光标位置
    EventSource     // 事件来源 (如攻击者)
}
```

## 技能生命周期

### 主动技能流程

```
玩家按键 → TriggerComponent.OnInput()
    │
    ├─ CanUseAbility()
    │   ├─ CooldownComponent.IsReady()
    │   ├─ ChargeComponent.HasCharge()
    │   └─ CostComponent.CanAfford()
    │
    ├─ ActivateAbility()
    │   ├─ CostComponent.ConsumeCost()
    │   ├─ ChargeComponent.ConsumeCharge()
    │   ├─ CooldownComponent.StartCooldown()
    │   └─ TargetingComponent.SelectTargets()
    │
    └─ AbilityEffect.Execute()
```

### 被动技能流程 (事件触发)

```
游戏事件发生 (如 Unit.Damaged)
    │
    ├─ TriggerComponent.OnEvent()
    │   ├─ 检查事件类型匹配
    │   └─ 检查触发概率 (如有)
    │
    ├─ CanTrigger()
    │   └─ CooldownComponent.IsReady() (内部冷却)
    │
    ├─ TriggerAbility()
    │   └─ CooldownComponent.StartCooldown()
    │
    └─ AbilityEffect.Execute()
        └─ 目标 = EventSource (攻击者)
```

### 被动技能流程 (周期触发)

```
TriggerComponent._Process(delta)
    │
    ├─ 更新周期计时器
    │
    ├─ 到达触发间隔?
    │   └─ AbilityEffect.Execute()
    │       └─ TargetingComponent.SelectTargets()
    │           └─ 范围内所有敌人
    │
    └─ 重置计时器
```

## 数据流设计

### 技能配置示例

**主动技能配置 (冲刺)**：
```csharp
{
    { DataKey.AbilityId, "dash" },
    { DataKey.AbilityName, "冲刺" },
    { DataKey.AbilityType, (int)AbilityType.Active },
    
    // 触发配置
    { DataKey.AbilityTriggerMode, (int)AbilityTriggerMode.Manual },
    
    // 冷却 + 充能
    { DataKey.AbilityCooldown, 3.0f },
    { DataKey.AbilityMaxCharges, 2 },
    { DataKey.AbilityChargeTime, 5.0f },
    
    // 效果参数
    { DataKey.AbilityRange, 150f },
    { DataKey.AbilityDuration, 0.2f }  // 无敌时间
}
```

**被动技能配置 (反伤)**：
```csharp
{
    { DataKey.AbilityId, "thorns" },
    { DataKey.AbilityName, "反伤" },
    { DataKey.AbilityType, (int)AbilityType.Passive },
    
    // 触发配置 - 事件触发
    { DataKey.AbilityTriggerMode, (int)AbilityTriggerMode.OnEvent },
    { DataKey.AbilityTriggerEvent, GameEventType.Unit.Damaged },
    
    // 内部冷却 (防止过于频繁)
    { DataKey.AbilityCooldown, 0.5f },
    
    // 效果参数
    { DataKey.AbilityDamage, 10f },
```

## 事件设计

| 事件类型 | 触发时机 | Payload 数据 |
|:---|:---|:---|
| `Ability.Added` | 技能被添加到单位 | `{Ability, Owner}` |
| `Ability.Removed` | 技能被移除 | `{Ability, Owner}` |
| `Ability.Activated` | 技能开始激活 | `{Ability, Owner, Targets}` |
| `Ability.Executed` | 技能效果执行完成 | `{Ability, Owner, Results}` |
| `Ability.Ready` | 技能冷却完成 | `{Ability, Owner}` |
| `Ability.LevelUp` | 技能升级 | `{Ability, OldLevel, NewLevel}` |

## 与其他系统的集成

### 与 DamageSystem 集成

技能造成伤害时，通过 `DamageService` 处理：

```csharp
var damageInfo = new DamageInfo
{
    Attacker = ability,
    Victim = target,
    BaseDamage = ability.Data.Get<float>(DataKey.AbilityDamage),
    DamageType = DamageType.Ability,
    Source = owner
};
DamageService.Instance.Process(damageInfo);
```

### 与 EntityManager 集成

技能召唤单位时，通过 `EntityManager.Spawn` 创建。

### Buff 系统说明

> [!NOTE]
> **Buff 将作为独立的 BuffEntity 系统实现**，不在技能 Component 中处理。
> 技能施加 Buff 时，将通过 BuffSystem (待实现) 创建 BuffEntity。

## 设计优势

1. **本质清晰**：主动/被动的区别在于触发方式，而非效果能力
2. **复用性强**：CooldownComponent 同时服务于主动冷却和被动内部冷却
3. **扩展性好**：TriggerComponent 支持多种触发模式，易于添加新模式
4. **职责分离**：每个组件职责单一，ChargeComponent 只服务于需要它的主动技能

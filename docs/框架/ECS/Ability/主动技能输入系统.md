## 1. 设计背景

### 1.1 独立游戏调研结论

| 游戏 | 主动技能数 | 输入设计 |
|:---|:---|:---|
| **Brotato (原版)** | 0 | 无主动技能，全自动射击 |
| **Vampire Survivors** | 0 | 同上 |
| **Dead Cells** | 2 | LT/RT 双技能槽 |
| **Hades** | 4 | Attack/Special/Cast/Call 分配到面部+肩键 |
| **塞尔达 BOTW** | 多 | D-Pad 分类 + 右摇杆轮盘选择 |

> [!IMPORTANT]
> **核心洞察**：Survivor 类游戏应尽量减少主动操作。主动技能最多 1~2 个常用 + 可选轮盘。

### 1.2 当前输入映射状态

```
D-Pad    → 已用于移动 (MoveLeft/Right/Up/Down)
BtnX     → 攻击键（可用于主动技能释放）
LB/RB    → 肩键（空闲，可用于技能切换/轮盘）
LT/RT    → 扳机（空闲）
```

## 2. 推荐方案

### 方案 A：单槽位 + LB/RB 切换（推荐）

```
┌────────────────────────────────────────┐
│  [当前技能图标]                        │
│                                        │
│  LB/RB → 切换主动技能（左/右循环）     │
│  X 键   → 释放当前选中技能             │
└────────────────────────────────────────┘
```

**优点**：
- 不占用移动键
- 单手可操作
- 支持无限技能数量

### 方案 B：LB 按住 + 右摇杆轮盘（塞尔达式）

```
┌────────────────────────────────────────┐
│  按住 LB → 显示技能轮盘               │
│  右摇杆指向 → 选择技能                │
│  释放 LB   → 确认选择并释放            │
└────────────────────────────────────────┘
```

**优点**：
- 视觉直观
- 支持大量技能

**缺点**：
- 需要 UI 支持
- 操作较复杂

## 3. 技术实现

### 3.1 触发方式

**推荐**：直接调用 `AbilitySystem.TryTriggerAbility()`

```csharp
**推荐**：使用 `ActiveSkillInputComponent` 组件封装输入逻辑，挂载到 `PlayerEntity`。

```csharp
// ActiveSkillInputComponent._Process()
if (InputManager.IsAttack())
{
    var ability = GetCurrentActiveAbility();
    var context = BuildCastContext(ability);
    
    // 直接调用 AbilitySystem，获取同步结果
    bool success = AbilitySystem.TryTriggerAbility(_entity, ability.Name, context);
    if (!success) PlayErrorSound();
}
```

**挂载方式**：
```csharp
// PlayerEntity._Ready()
EntityManager.AddComponent(this, new ActiveSkillInputComponent());
```

**理由**：
1. **解耦**：输入逻辑与实体逻辑分离，符合组合优于继承原则
2. **同步反馈**：保留 `TryTriggerAbility` 的直接调用优势
3. **职责单一**：PlayerEntity 仅负责组装组件

### 3.2 目标解析

| AbilityTargetSelection | 处理方式 |
| :--- | :--- |
| `None` | 无需处理 |
| `Unit` | TargetSelector 智能索敌（最近敌人） |
| `Point` | **右摇杆方向** × AbilityRange |
| `UnitOrPoint` | 优先索敌，无目标则右摇杆方向释放 |

### 3.3 AbilitySystem 严格模式

AbilitySystem 保持**严格验证**：若配置需要 Unit/Point 而上下文缺失，返回 false 并记录错误日志。

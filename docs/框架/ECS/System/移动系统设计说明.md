# 移动系统设计说明

## 一、系统定位

移动系统是**通用 Entity 运动执行层**，负责"怎么移动"，不负责"为什么移动"。

- **业务层**（技能/AI/EffectTool）：决定用哪种运动模式，构建 `MovementParams` 并通过事件传入
- **移动系统**（EntityMovementComponent + IMovementStrategy）：持有参数、执行运动、更新位置/朝向
- **适用实体**：`Node2D + IEntity` 与 `CharacterBody2D + IEntity`

## 二、核心架构

### 参数传递方式

运动参数不再写入 DataKey，而是通过事件携带的 `MovementParams` 结构体传入：

```csharp
entity.Events.Emit(
    GameEventType.Unit.MovementStarted,
    new GameEventType.Unit.MovementStartedEventData(MoveMode.TargetPoint, new MovementParams
    {
        Mode        = MoveMode.TargetPoint,
        TargetPoint = new Vector2(900, 360),
        MaxDistance = 300f,          // 可选
        DestroyOnComplete = true,    // 可选
    }));
```

`EntityMovementComponent` 将 `MovementParams` 存为私有字段，策略通过 `in MovementParams` 参数只读访问。

### 策略实例化

每次切换模式时，注册表以工厂函数**新建策略实例**，不再复用单例：

- 每次运动拥有独立策略实例，策略可安全持有私有运行时状态（如 `_currentAngle`、`_startPoint`、`_lengthLut` 等）
- 运动结束后实例被丢弃，无需手动清理状态

### 调度器结构

```text
EntityMovementComponent（调度器）
  ├── _params: MovementParams       -- 本次运动输入参数
  ├── _elapsedTime / _traveledDistance / _moveCompleted  -- 统计私有字段
  ├── 注册时按 DataKey.DefaultMoveMode 初始化默认策略
  ├── 监听 MovementStarted 事件切换策略（携带新 MovementParams）
  ├── RunMovementLogic()：委托策略 Update(entity, data, delta, elapsedTime, in params)
  ├── _Process / _PhysicsProcess：由策略 UsePhysicsProcess 决定走哪路
  ├── AccumulateTravel()：累计时间/距离到私有字段
  └── CheckEndConditions()：读 _params.MaxDuration / MaxDistance
```

### 关键组件

| 组件 | 职责 |
|------|------|
| `MovementParams` | 本次运动所有输入参数（`record struct`，`init` 属性） |
| `IMovementStrategy` | 策略接口：`OnEnter / Update(in MovementParams) / OnExit` |
| `MovementStrategyRegistry` | `MoveMode → Func<IMovementStrategy>` 工厂注册表 |
| `MovementHelper` | 朝向更新、到达距离、环绕步进等策略共用工具 |
| `VelocityResolver` | Velocity 分层合成 |

## 三、13 种运动模式

| MoveMode | 策略类 | 核心行为 | 关键 MovementParams 字段 |
|---|---|---|---|
| None (0) | — | 不更新位置 | — |
| FixedDirection (1) | FixedDirectionStrategy | 沿 DataKey.Velocity 直线前进 | MaxDistance/MaxDuration |
| TargetPoint (2) | TargetPointStrategy | 向目标点直线冲锋，到达后完成 | TargetPoint, ReachDistance |
| TargetEntity (3) | TargetEntityStrategy | 追踪目标实体，丢失后直线继续 | TargetNode, ReachDistance |
| OrbitPoint (4) | OrbitPointStrategy | 围绕固定点圆周运动 | OrbitCenter, OrbitRadius, OrbitAngularSpeed |
| OrbitEntity (5) | OrbitEntityStrategy | 围绕目标实体动态环绕 | TargetNode, OrbitRadius, OrbitAngularSpeed |
| Spiral (6) | SpiralStrategy | 螺旋运动（半径渐变） | OrbitCenter, OrbitRadius, OrbitTargetRadius, OrbitAngularSpeed |
| SineWave (7) | SineWaveStrategy | 正弦波形前进 | WaveAmplitude, WaveFrequency（+DataKey.Velocity 初始方向） |
| BezierCurve (8) | BezierCurveStrategy | N 阶贝塞尔曲线，支持匀速 | BezierPoints, BezierDuration |
| Boomerang (9) | BoomerangStrategy | 去程→可选停顿→回程 | TargetPoint, BoomerangPauseTime |
| AttachToHost (10) | AttachToHostStrategy | 附着跟随宿主位置 + 偏移 | TargetNode（+DataKey.EffectOffset） |
| PlayerInput (11) | PlayerInputStrategy | 玩家输入驱动（DefaultMoveMode） | 无，读 DataKey.MoveSpeed/Acceleration |
| AIControlled (12) | AIControlledStrategy | AI 意图驱动（DefaultMoveMode） | 无，读 DataKey.AIMoveDirection 等 |

> 所有模式对 Node2D/Area2D 和 CharacterBody2D 通用。策略只写 `DataKey.Velocity`，调度器根据节点类型统一执行位移。

## 四、Velocity 分层合成

`VelocityResolver.Resolve(data)` 优先级从高到低：

```text
1. IsMovementLocked = true  → Vector2.Zero（完全锁定）
2. VelocityOverride ≠ Zero  → VelocityOverride（击退/硬控覆盖）
3. 否则                      → Velocity + VelocityImpulse（基础 + 单帧冲量）
```

## 五、结束条件

### MovementParams 通用结束条件

| 字段 | 语义 | 规则 |
|---|---|---|
| `MaxDuration` | 最大持续时间（秒） | -1=不限制 |
| `MaxDistance` | 最大移动距离（像素） | -1=不限制 |
| `DestroyOnComplete` | 完成后销毁实体 | false=发事件后回退默认模式 |

### 策略主动完成

策略 `Update()` 返回 `MovementUpdateResult.Complete()` 即主动完成（如到达目标点、宿主消失）。

### 完成事件

```csharp
// MovementCompletedEventData 直接携带统计数据，无需读 DataKey
entity.Events.On<GameEventType.Unit.MovementCompletedEventData>(
    GameEventType.Unit.MovementCompleted,
    evt => _log.Info($"完成 Mode={evt.Mode}, 历时={evt.ElapsedTime:F2}s, 距离={evt.TraveledDistance:F1}px"));
```

## 六、AttachToHost 附着跟随

```text
EffectComponent.SetupAttachment()
  → entity.Events.Emit(MovementStarted, new MovementStartedEventData(
        MoveMode.AttachToHost,
        new MovementParams { Mode = MoveMode.AttachToHost, TargetNode = host2D }))
  → EntityMovementComponent 切入 AttachToHostStrategy
    → 每帧对齐到 TargetNode.GlobalPosition + DataKey.EffectOffset
    → 宿主失效时返回 Complete
```

## 七、DataKey 速查（精简后仅 7 个跨系统共享键）

| DataKey | 类型 | 说明 |
|---|---|---|
| `Velocity` | Vector2 | 策略写入的基础意图速度 |
| `VelocityOverride` | Vector2 | 击退/控制技能覆盖速度 |
| `VelocityImpulse` | Vector2 | 单帧冲量（用后自动清零） |
| `IsMovementLocked` | bool | 眩晕/冻结锁定开关 |
| `Acceleration` | float | PlayerInputStrategy 读取的平滑系数 |
| `MoveMode` | MoveMode | 当前运动模式（组件写入，外部可观察） |
| `DefaultMoveMode` | MoveMode | 默认模式（Entity 初始化写入） |

> 其他运动参数（目标点、速度、振幅、控制点等）均已迁移到 `MovementParams`，不再占用 DataKey。

## 八、扩展新运动模式

1. 在 `MovementEnums.cs` 添加 `MoveMode` 枚举值
2. 在 `MovementParams` 添加所需字段（`init` 属性 + 合理默认值）
3. 创建策略类实现 `IMovementStrategy`，私有字段存运行时状态
4. 用 `[ModuleInitializer]` 注册工厂函数到 `MovementStrategyRegistry`
5. 补全策略类头注释（描述 + 使用示例 + 典型用途）

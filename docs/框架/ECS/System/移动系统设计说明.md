# 移动系统设计说明

## 一、系统定位

移动系统是**通用 Entity 运动执行层**，负责"怎么移动"，不负责"为什么移动"。

- **业务层**（技能/AI/EffectTool）：决定用哪种运动模式，写入参数到 Data
- **移动系统**（EntityMovementComponent + IMovementStrategy）：执行参数，更新位置/朝向
- **适用实体**：`Node2D + IEntity` 与 `CharacterBody2D + IEntity`

## 二、核心架构

### 策略模式

```text
EntityMovementComponent（调度器，挂载在 Entity 的 Component 节点下）
  ├── 每帧检测 DataKey.MoveMode 变化
  │   ├── 切换时：OnExit(旧策略) → OnEnter(新策略)
  │   └── 无变化：继续当前策略
  ├── RunMovementLogic()：委托策略计算运动意图（策略只写 DataKey.Velocity）
  ├── Node2D / Area2D 路径：_Process()
  │   └── ApplyNodeMovement()：GlobalPosition += Velocity * delta + 朝向旋转
  ├── CharacterBody2D 路径：_PhysicsProcess()
  │   ├── VelocityResolver.Resolve()（分层合成）
  │   └── MoveAndSlide() + 同步速度 + 朝向翻转
  ├── AccumulateTravel()：累计时间和距离
  └── CheckEndConditions()：MoveMaxDuration / MoveMaxDistance
```

### 关键组件

| 组件 | 职责 |
|------|------|
| `IMovementStrategy` | 策略接口：Update（纯计算，只写 Velocity）/ OnEnter / OnExit |
| `MovementStrategyRegistry` | MoveMode → Strategy 的静态映射 |
| `MovementHelper` | 朝向旋转、到达距离等策略共用方法 |
| `VelocityResolver` | Velocity 分层合成（解决多组件写入冲突） |

### 自注册机制

策略通过 `[ModuleInitializer]` 在程序启动时自动注册到 `MovementStrategyRegistry`，无需手动配置。

## 三、12 种运动模式

| MoveMode (枚举值) | 策略类 | 核心行为 |
|---|---|---|
| None (0) | — | 不更新位置 |
| FixedDirection (1) | FixedDirectionStrategy | 沿 Velocity 方向匀速直线 |
| TargetPoint (2) | TargetPointStrategy | 向目标点冲锋，到达后完成 |
| TargetEntity (3) | TargetEntityStrategy | 追踪目标实体，丢失后直线继续 |
| OrbitPoint (4) | OrbitPointStrategy | 围绕固定点圆周运动 |
| OrbitEntity (5) | OrbitEntityStrategy | 围绕目标实体动态环绕 |
| Spiral (6) | SpiralStrategy | 螺旋运动（半径从 OrbitRadius 渐变至 OrbitTargetRadius） |
| SineWave (7) | SineWaveStrategy | 正弦波形前进（振幅/频率/相位可配） |
| BezierCurve (8) | BezierCurveStrategy | N 阶贝塞尔曲线，支持匀速模式 |
| Boomerang (9) | BoomerangStrategy | 去程→可选停顿→回程 |
| AttachToHost (10) | AttachToHostStrategy | 附着跟随宿主位置 + 偏移 |
| PlayerInput (11) | PlayerInputStrategy | 玩家输入驱动速度 |
| AIControlled (12) | AIControlledStrategy | AI 移动意图驱动速度 |

> 所有 12 种模式对 Node2D/Area2D 和 CharacterBody2D 通用。策略只写 `DataKey.Velocity`，调度器根据节点类型统一执行位移。

## 四、Velocity 分层合成

`VelocityResolver.Resolve(data)` 解决玩家/敌人/技能多组件同时写入 `DataKey.Velocity` 的冲突：

```text
优先级从高到低：
1. IsMovementLocked = true  → 返回 Vector2.Zero（完全锁定）
2. VelocityOverride ≠ Zero  → 返回 VelocityOverride（技能/Dash 覆盖）
3. 否则                      → 返回 Velocity + VelocityImpulse（基础 + 脉冲叠加）
```

**已接入**：`EntityMovementComponent` 的 CharacterBody2D 路径（统一玩家/敌人物理移动）。

AI 行为树动作节点在非 `MoveMode.AIControlled` 模式下会暂停写入 `AIMoveDirection/AIMoveSpeedMultiplier`，避免覆盖击退/冲刺等特殊移动模式。

## 五、结束条件

### 通用结束条件（调度器统一处理）

| DataKey | 语义 | 规则 |
|---|---|---|
| MoveMaxDuration | 最大持续时间（秒） | -1=不限制，≥0 时参与比较 |
| MoveMaxDistance | 最大移动距离（像素） | -1=不限制，≥0 时参与比较 |

### 策略主动完成

策略 `Update()` 返回 -1 表示运动完成（如 TargetPoint 到达目标、AttachToHost 宿主消失）。

### 完成后行为

- `MoveDestroyOnComplete = true`：自动销毁实体
- `MoveDestroyOnComplete = false`（默认）：仅设置 `MoveCompleted = true`，发出 `MovementCompleted` 事件

## 六、附着跟随（AttachToHost）

### 设计背景

特效附着跟随（`EffectComponent._Process` 中直接设置 `GlobalPosition`）本质是一种移动模式。
重构后由移动系统统一管理，消除了 `EntityMovementComponent` 对 `EffectIsAttached` 的硬编码跳过。

### 工作流程

```text
EffectTool.Spawn(附着模式)
  → EffectComponent.SetupAttachment()
    → Data.Set(MoveTargetNode, 宿主Node2D)
    → Data.Set(MoveMode, AttachToHost)
  → EntityMovementComponent 检测到 MoveMode=AttachToHost
    → 委托 AttachToHostStrategy.Update()
      → selfNode.GlobalPosition = hostNode.GlobalPosition + Offset
      → 宿主无效时返回 -1（触发运动完成）
```

### 职责分离

- **AttachToHostStrategy**：纯位置跟随（每帧更新 GlobalPosition）
- **EffectComponent**：宿主销毁监听、生命周期管理、动画播放（不再参与位置更新）

## 七、数据键速查

### 通用参数（DataCategory_Movement.Basic）

| DataKey | 类型 | 默认值 | 说明 |
|---|---|---|---|
| MoveMode | MoveMode | None | 运动模式 |
| MoveMaxDuration | float | -1 | 最大持续时间（-1=不限制） |
| MoveMaxDistance | float | -1 | 最大移动距离（-1=不限制） |
| MoveElapsedTime | float | 0 | 已运动时间（运行时） |
| MoveTraveledDistance | float | 0 | 已移动距离（运行时） |
| MoveCompleted | bool | false | 运动是否完成 |
| MoveDestroyOnComplete | bool | false | 完成后是否销毁 |
| RotateToVelocity | bool | false | 自动朝向速度方向 |

### 目标参数（DataCategory_Movement.Target）

| DataKey | 类型 | 说明 |
|---|---|---|
| MoveTargetPoint | Vector2 | 目标点坐标 |
| MoveTargetNode | string(const) | 目标 Node2D 引用 |
| MoveReachDistance | float | 到达距离阈值（默认 5） |

### 环绕/螺旋参数（DataCategory_Movement.Orbit）

OrbitCenterPoint / OrbitRadius / OrbitTargetRadius / OrbitAngle / OrbitAngularSpeed / OrbitRadialSpeed / OrbitClockwise

### 波形参数（DataCategory_Movement.Wave）

WaveAmplitude / WaveFrequency / WavePhase

### 贝塞尔参数（DataCategory_Movement.Bezier）

BezierControlPoints / BezierControlPoint1 / BezierControlPoint2 / BezierStartPoint / BezierDuration / BezierUniformSpeed / BezierLengthLut

### 回旋镖参数（DataCategory_Movement.Boomerang）

BoomerangStartPoint / BoomerangReturning / BoomerangPauseTime / BoomerangPauseTimer

## 八、扩展新运动模式

1. 在 `Data/DataKey/Component/Movement/MovementEnums.cs` 添加 `MoveMode` 枚举值
2. 创建策略类实现 `IMovementStrategy`（OnEnter/Update/OnExit）
3. 添加 `[ModuleInitializer]` 静态方法注册到 `MovementStrategyRegistry`
4. 如需新参数在 `Data/DataKey/Component/Movement/DataKey_Movement.cs` 添加 DataKey

**⚠️ 策略类是单例**：注册表全局共享实例，禁止持有实例级业务状态字段，所有运行时状态必须存入 `Data`。

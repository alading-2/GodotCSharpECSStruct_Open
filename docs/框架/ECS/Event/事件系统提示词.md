# 事件系统 (Event System) 重构设计与提示词

## 1. 核心问题分析

### 当前现状 (`EventBus.cs`)
- **实现方式**: 纯静态类，基于 C# 原生 `event`。
- **优点**: 类型安全，编译期检查，性能高。
- **缺点**: 
  - **耦合度高**: 新增事件必须修改 `EventBus` 类定义。
  - **扩展性差**: 无法支持动态事件类型。
  - **不支持局部作用域**: 只有全局单例，无法实现 Entity 内部或 Component 之间的局部通信。

### 参考目标 (`EventBus.ts`)
- **实现方式**: 实例类 + 单例模式，基于 `Map<string, subscription[]>`。
- **特性**:
  - 支持 `on`, `off`, `emit`, `once` 动态接口。
  - 支持 优先级 (`priority`)。
  - 支持 字符串事件 Key，解耦发布者和订阅者。
  - 支持 泛型数据传递。

## 2. 架构定位与选型

### ❓ 问题：Event 属于 Tool 还是 System？
**回答**: Event 既不是 Tool 也不是 System。
- **不是 Tool**: Tool 通常指无状态的静态工具库 (如 `MathUtil`, `DataUtils`)。EventBus 是高度**有状态**的（存储了订阅关系）。
- **不是 System**: ECS 的 System 指的是每帧执行逻辑的处理器 (`_Process`)。EventBus 是**基础设施 (Infrastructure)** 或 **核心机制 (Core Mechanism)**。

### 📂 推荐路径
`Src/ECS/Event/`  
(保持在 ECS 目录下，因为它是 ECS 实体间通信的血脉)

## 3. 重构设计方案

### 3.1 核心类设计

我们需要一个通用的与实现无关的事件总线类，暂定名为 `GameEventBus` (或 `LocalEventBus`)。

#### `GameEventBus` (类)
可实例化的事件总线，用于 Entity 或局部系统。

**API 设计 (参考 TypeScript 版本但适配 C#)**:
```csharp
public class GameEventBus
{
    // 订阅
    public void On<T>(string eventName, Action<T> handler, int priority = 0);
    public void On(string eventName, Action handler, int priority = 0); // 无参版本
    
    // 一次性订阅
    public void Once<T>(string eventName, Action<T> handler, int priority = 0);

    // 取消订阅
    public void Off<T>(string eventName, Action<T> handler);
    public void Off(string eventName, Action handler);

    // 触发
    public void Emit<T>(string eventName, T data);
    public void Emit(string eventName); // 无参版本

    // 清理
    public void Clear();
}
```

### 3.2 接入方式

#### 全局事件 (`GlobalEventBus`)
保留 `EventBus` 的全局访问便利性，但底层替换为 `GameEventBus` 实例，或者保留静态 Event 作为一个特定层面的封装。
**建议**: 创建 `GlobalEvent` 静态类，内部持有一个 `GameEventBus.Global` 单例，用于跨模块的大型事件（如 `GameStart`, `LevelLoaded`）。

#### Entity 事件 (`Entity.Events`)
在 `Entity` (或 `IEntity`) 中集成事件总线，实现组件间通信。

```csharp
// 在 Entity 类中
public partial class Entity : Node, IEntity 
{
    // 每个实体拥有独立的事件总线
    public GameEventBus Events { get; } = new GameEventBus(); 
    
    public override void _ExitTree()
    {
        Events.Clear(); // 自动清理防止内存泄漏
        base._ExitTree();
    }
}
```

**使用场景**:
- `HealthComponent` 扣血后 -> `entity.Events.Emit("Damage", new DamageInfo(...))`
- `AnimationComponent` 监听 -> `entity.Events.On<DamageInfo>("Damage", PlayHurtAnim)`
- `AchievementSystem` 监听全局 -> `GlobalEvent.On("EnemyDied", ...)`

## 4. 实施提示词 (Prompt)

以下是将用于生成代码的提示词说明：

---

**任务**: 基于 `参考/ECS/EventSystem/EventBus.ts` 的逻辑，用 C# 在 `Src/ECS/Event/` 下重构事件系统。

**要求**:
1.  **创建 `GameEventBus` 类**: 
    -   实现基于 `Dictionary<string, List<Subscription>>` 的事件管理。
    -   支持泛型 `Action<T>` 和无参 `Action`。
    -   支持优先级排序 (Priority)。
    -   支持 `Once` (触发一次后自动移除)。
    -   需要处理订阅列表的线程安全或遍历时修改集合的问题 (即 Emit 时如果 Off 导致的异常)。
    -   使用 `Log` 系统打印调试信息。

2.  **集成到 Entity**:
    -   修改或扩展 Entity 机制，使每个 Entity 实例都能访问自己的 `Events` 对象。
    -   确保 Entity 销毁时自动清空其 EventBus。

3.  **重构 Global Event**:
    -   新建 `GlobalEvent` 类 (或利用 `EventBus.cs`) 作为一个封装了 `GameEventBus` 的静态访问点。
    -   逐步替换原有的硬编码 `event Action` (可先共存，逐步标记 Obsolete)。

4.  **代码风格**:
    -   C# 10/11 特性。
    -   完备的注释。
    -   命名规范遵循项目规则 (PascalCase)。

**注意事项**:
-   TypeScript 的 `any` 在 C# 中应尽量通过泛型 `T` 解决，避免过多的装箱拆箱。
-   C# 的 `Delegate` 比较不像 JS 的 Function 那么灵活，需要区分 `Action` 和 `Action<T>`，或者定义统一的接口。
-   防止内存泄漏是关键，必须提供方便的 `Off` 或自动清理机制。

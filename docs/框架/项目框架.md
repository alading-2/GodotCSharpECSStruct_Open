# Brotato 复刻项目架构指南

## 1. 架构选型与核心理念

对于《土豆兄弟》(Brotato) 这类 Survivor-like (割草类) 游戏，核心技术挑战在于处理同屏大量的动态实体（敌人、子弹、掉落物）。

### 推荐架构：Godot 高性能混合架构

我们**不采用**传统的第三方 ECS 框架，而是利用 Godot 4.x 强大的原生特性进行“混合式”开发：

1. **逻辑层 (Logic)**: 使用 **节点组合 (Node Composition)** 模式。即“伪 ECS”，将功能封装为独立的 Node 组件（如 `HealthComponent`），组合在实体上。
2. **数据层 (Data)**: 使用 **Resource (资源)** 进行数据驱动开发。所有属性（攻击力、怪物数值、波次配置）都应是 `.tres` 文件。
3. **优化层 (Optimization)**:
   - **中等负载**: 强制使用 **对象池 (Object Pooling)** 复用节点。
   - **高负载 (2000+ 实体)**: 对于子弹和伤害数字，使用 `RenderingServer` 和 `PhysicsServer2D` 进行底层托管。

## 3. 关键系统实现指南

### 3.1 组件化系统 (The "Godot ECS" approach)

不要让 `Player.cs` 或 `Enemy.cs` 继承同一个巨大的 `Entity` 类。相反，使用组合。

- **HealthComponent (Node)**
  - 职责：存储 `MaxHp`, `CurrentHp`，处理受伤逻辑，发出 `Died` 信号。
  - 用法：玩家和敌人都挂载此节点。
- **HurtboxComponent (Area2D)**
  - 职责：定义受击范围，检测碰撞。
  - 连接：当检测到 `Hitbox` 进入，调用父节点的 `HealthComponent.TakeDamage()`。
- **VelocityComponent (Node)**
  - 职责：处理移动加速度、摩擦力，控制父节点的 `CharacterBody2D`。

### 3.2 数据驱动 (Resource-based Design)

避免硬编码（Hard-coding）。例如，不要在代码里写 `damage = 10`。

创建自定义资源类：

```csharp
// WeaponData.cs (继承自 Resource)
using Godot;

[GlobalClass]
public partial class WeaponData : Resource
{
    [Export] public string Name { get; set; }
    [Export] public Texture2D Icon { get; set; }
    [Export] public float Damage { get; set; }
    [Export] public float Cooldown { get; set; }
    [Export] public PackedScene ProjectileScene { get; set; }
}
```

然后在编辑器中右键新建 Resource -> WeaponData，填入数值。这使得非程序员（策划）也能调整游戏平衡。

### 3.3 对象池 (Object Pooling)

**这是 Survivor 类游戏的性能生命线。**

- **原理**: 游戏开始时预先实例化 500 个子弹和 100 个敌人，并隐藏它们。
- **生成**: 需要生成时，从池中取出一个隐藏的实例，重置其位置和状态，设为可见。
- **销毁**: 当物体“死亡”或飞出屏幕时，不要 `queue_free()`，而是 `hide()` 并放回池中。

## 4. 进阶优化：Server API (针对海量物体)

当常规节点（Node）数量超过 1000 时，可能会掉帧。此时应针对特定高频物体使用 Server API。

**场景**: 玩家发射的加特林子弹、满屏的经验球。

**方案**:

- 不使用 `Sprite2D` 节点，而是使用 `RenderingServer` 直接向 GPU 发送绘图指令。
- 不使用 `Area2D` 节点，而是使用 `PhysicsServer2D` 进行纯数学碰撞检测。

_注意：优先开发原型，仅在性能分析器 (Profiler) 显示瓶颈时再进行此步优化。_

## 5. 建议开发顺序

1. **核心战斗原型**:
   - 玩家移动 + 一把枪 + 一个靶子敌人。
   - 实现 `HealthComponent` 和 `Hitbox/Hurtbox` 交互。
2. **对象池集成**:
   - 实现子弹的发射和回收。
3. **波次管理 (Wave Manager)**:
   - 实现定时刷新敌人的逻辑。
4. **数据层构建**:
   - 定义 `Weapon` 和 `Enemy` 的 Resource，并制作几个不同的变体。
5. **UI 与 循环**:
   - HUD 显示血量/经验。
   - 胜利/失败判定。

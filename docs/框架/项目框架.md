## 1. 架构概览

### 1.1 全局单例管理器 (AutoLoad)

**核心职责**: 统一管理所有全局单例（Manager/Service），解决 Godot 默认 AutoLoad 无法控制加载顺序、依赖关系和类型安全访问的问题。

- **加载优先级**: 通过 `Priority` 常量定义（Core: 0, System: 100, Game: 200, Debug: 900），确保底层工具（如日志）最先启动。
- **依赖检查**: 支持声明依赖项，若依赖的单例未注册或加载失败，将抛出错误。
- **类型安全**: 提供泛型 `Get<T>()` 接口，无需手动转型。
- **ECS 桥梁**: 作为 ECS 系统层（System/Service）的容器，所有无状态系统均在此注册并由逻辑层调用。

**代码示例 (注册)**:

```csharp
private void Configure()
{
    // 核心工具
    Register("LogWriter", "res://Src/Tools/Logger/LogWriter.cs", Priority.Core);

    // ECS 系统服务
    Register("DamageCalculationService", "res://Src/ECS/System/DamageCalculationService.cs", Priority.System);
    Register("EffectSystem", "res://Src/ECS/System/EffectSystem.cs", Priority.System, new[] { "DamageCalculationService" });
}
```

**代码示例 (调用)**:

```csharp
// 在任何地方获取系统
var damageService = AutoLoad.Instance.Get<DamageCalculationService>();
```

### 1.2 核心理念

**目标游戏类型**: Survivor-like 割草游戏（类似《土豆兄弟》）
**核心挑战**: 同屏处理 500+ 动态实体（敌人、子弹、掉落物）

### 1.3 架构选型：Godot C# 伪 ECS 混合架构

**为什么不用传统 ECS 框架？**

- Godot 的 Node 系统已经提供了强大的组合能力
- 第三方 ECS 框架与 Godot 编辑器集成困难
- 伪 ECS 可以充分利用 Godot 的可视化编辑器

**三层架构设计**:

```
┌─────────────────────────────────────────────────────┐
│  逻辑层 (Logic Layer)                                │
│  - Node Composition (组件组合)                       │
│  - Component System (HealthComponent, etc.)         │
└─────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────┐
│  数据层 (Data Layer)                                 │
│  - 静态配置: Resource (.tres 文件)                   │
│  - 动态数据: Data 容器 (挂载到 Node)                 │
└─────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────┐
│  优化层 (Optimization Layer)                         │
│  - ObjectPool (对象池复用)                           │
│  - RenderingServer/PhysicsServer (高负载场景)       │
└─────────────────────────────────────────────────────┘
```

### ✅ 2.5 Entity System (实体系统)

**路径**: `Src/ECS/Entity/Core/`

**核心价值**:

- **EntityManager**: 统一的实体生命周期管理入口（生成、注册、查询、回收）
- **EntityRelationshipManager**: 高效的实体关系管理（三索引结构）
- **Scene 即 Entity**: 完全符合 Godot 设计哲学，无需 Entity 基类

**设计亮点**:

```csharp
// 一行代码生成 Entity（自动：对象池获取 + 数据注入 + 注册）
var enemy = EntityManager.Spawn<Enemy>(new EntitySpawnConfig
{
    Resource = enemyResource,
    UsingObjectPool = true,
    PoolName = ObjectPoolNames.EnemyPool,
    Position = position
});

// 建立关系（玩家拾取物品）
EntityRelationshipManager.AddRelationship(
    playerId, itemId, EntityRelationshipType.UNIT_TO_ITEM
);

// 查询关系（获取玩家的所有物品）
var itemIds = EntityRelationshipManager.GetChildEntitiesByParentAndType(
    playerId, EntityRelationshipType.UNIT_TO_ITEM
);

// 回收 Entity（自动清理关系 + 归还对象池）
EntityManager.Recycle(enemy);
```

**核心特性**:

1. **三个 Spawn 重载**：

   - `Spawn<T>(Resource, Node?)` - 无位置（Buff、背包物品）
   - `Spawn<T>(Resource, Vector2, Node?)` - 带位置（Enemy、掉落物）
   - `Spawn<T>(Resource, Vector2, float, Node?)` - 带方向（Bullet、投射物）

2. **自动化流程**：

   ```
   EntityManager.Spawn()
       ↓
   ObjectPool.Get()        # 从对象池获取实例
       ↓
   InjectResourceData()    # 注入 Resource 配置到 Data
       ↓
   Register()              # 自动注册到 EntityManager
       ↓
   返回已配置好的实例
   ```

3. **三索引关系管理**：

   - **主存储**：`relationshipId -> RelationshipRecord`（完整关系记录）
   - **父索引**：`parentEntityId -> Set<relationshipId>`（快速查询子 Entity）
   - **子索引**：`childEntityId -> Set<relationshipId>`（快速查询父 Entity）
   - **类型索引**：`relationType -> Set<relationshipId>`（按类型查询）

4. **智能回收**：
   ```csharp
   // Recycle 自动完成：
   // 1. 清理所有关系（父子关系、装备关系等）
   // 2. 从 EntityManager 注销
   // 3. 归还到对象池
   EntityManager.Recycle(entity);
   ```

**使用场景**:

- **SpawnSystem 生成敌人**：`EntityManager.Spawn<Enemy>(new EntitySpawnConfig { Resource = enemyRes, UsingObjectPool = true, PoolName = ObjectPoolNames.EnemyPool, Position = pos })`
- **玩家拾取物品**：`EntityRelationshipManager.AddRelationship(playerId, itemId, UNIT_TO_ITEM)`
- **武器装备系统**：`EntityRelationshipManager.AddRelationship(unitId, weaponId, UNIT_TO_WEAPON)`
- **Buff/Debuff 管理**：`EntityRelationshipManager.AddRelationship(unitId, buffId, UNIT_TO_BUFF)`
- **Entity 销毁**：`EntityManager.Recycle(entity)` 而不是 `QueueFree()`

**架构意义**:

这是伪 ECS 的核心管理层，实现了：

- **统一生命周期管理**：避免手动 Instantiate/QueueFree
- **对象池集成**：自动复用实例，提升性能
- **数据驱动**：Resource 配置自动注入到 Data
- **关系管理**：支持复杂的 Entity 间关系（装备、拥有、Buff 等）

**详细文档**: `Docs/框架/ECS/Entity/README.md`

#### 必需组件列表

**1. HealthComponent (生命值组件)**

```csharp
using System;

public partial class HealthComponent : Node
{
    [Export] public float MaxHp { get; set; } = 100;
    public float CurrentHp { get; private set; }

    // ✅ 使用 C# 原生 Event（推荐）
    public event Action<float>? Damaged;   // 参数：伤害值
    public event Action<float>? Healed;    // 参数：治疗量
    public event Action? Died;             // 无参数

    public override void _Ready()
    {
        CurrentHp = MaxHp;
    }

    public void TakeDamage(float damage)
    {
        if (CurrentHp <= 0) return;

        CurrentHp = Math.Max(0, CurrentHp - damage);
        Damaged?.Invoke(damage);

        if (CurrentHp <= 0)
        {
            Died?.Invoke();
        }
    }

    public void Heal(float amount)
    {
        if (CurrentHp >= MaxHp) return;

        CurrentHp = Math.Min(MaxHp, CurrentHp + amount);
        Healed?.Invoke(amount);
    }
}
```

**使用场景**: 玩家、敌人、可破坏物体

**为什么用 C# Event？**

- ✅ 性能更好（编译期优化）
- ✅ 类型安全（编译期检查）
- ✅ 不依赖 Godot 引擎（可用于纯 C# 类）
- ✅ 代码更简洁

```csharp
public partial class VelocityComponent : Node

    public float Speed => _data.Get<float>("Speed", 400f);

        _parent.MoveAndSlide();
    }
}
```

```csharp
public static class DataResourceIndex
```

### 🔲 3.3 Event System (事件系统)

**优先级**: ⭐⭐⭐⭐⭐ (最高)

**职责**: 全局事件总线，解耦模块间通信

**核心原则**: **优先使用 C# 原生 Event，仅在必要时使用 Godot Signal**

#### 为什么优先用 C# Event？

| 特性         | C# Event              | Godot Signal              |
| :----------- | :-------------------- | :------------------------ |
| **性能**     | ✅ 极快（编译期优化） | ❌ 较慢（运行时反射）     |
| **类型安全** | ✅ 编译期检查         | ❌ 运行时检查             |
| **依赖性**   | ✅ 纯 C# 可用         | ❌ 必须继承 Node          |
| **适用场景** | ✅ 内部架构、逻辑解耦 | ⚠️ UI 连线、GDScript 交互 |

#### 实现方案：静态事件总线（推荐）

```csharp
// Src/ECS/Event/EventBus.cs
using System;
using Godot;

/// <summary>
/// 全局事件总线 - 使用 C# 原生 Event
/// 无需继承 Node，可以是纯静态类
/// </summary>
public static class EventBus
{
    // === 战斗事件 ===
    public static event Action<Node, Vector2>? EnemyDied;      // 参数：敌人节点, 位置
    public static event Action<float>? PlayerDamaged;          // 参数：伤害值
    public static event Action<float>? PlayerHealed;           // 参数：治疗量

    // === 波次事件 ===
    public static event Action<int>? WaveStarted;              // 参数：波次编号
    public static event Action<int>? WaveCompleted;            // 参数：波次编号

    // === 经验/升级事件 ===
    public static event Action<int>? ExperienceGained;         // 参数：经验值
    public static event Action<int>? LevelUp;                  // 参数：新等级
    public static event Action<UpgradeData>? UpgradeSelected;  // 参数：升级数据

    // === 游戏状态事件 ===
    public static event Action? GamePaused;
    public static event Action? GameResumed;
    public static event Action<bool>? GameOver;                // 参数：是否胜利

    // === 便捷触发方法（可选） ===
    public static void TriggerEnemyDied(Node enemy, Vector2 position)
    {
        EnemyDied?.Invoke(enemy, position);
    }

    public static void TriggerPlayerDamaged(float damage)
    {
        PlayerDamaged?.Invoke(damage);
    }

    // ... 其他触发方法
}
```

#### 使用示例

**发布事件**:

```csharp
// 方式1：直接触发
EventBus.EnemyDied?.Invoke(enemy, enemy.GlobalPosition);

// 方式2：使用便捷方法
EventBus.TriggerEnemyDied(enemy, enemy.GlobalPosition);
```

**订阅事件**:

```csharp
public partial class WaveSystem : Node
{
    public override void _Ready()
    {
        // 订阅
        EventBus.EnemyDied += OnEnemyDied;
    }

    private void OnEnemyDied(Node enemy, Vector2 position)
    {
```

```csharp
        // 暴击计算
        if (attacker.HasData())
        {
            float critChance = attacker.GetData().Get<float>("CritChance", 0);
            float critMultiplier = attacker.GetData().Get<float>("CritMultiplier", 2.0f);

                ShowCritEffect(target);

        // 伤害减免
        if (target.HasData())
        {
            float damageReduction = target.GetData().Get<float>("DamageReduction", 0);
            finalDamage *= (1 - damageReduction);
        }

        return finalDamage;
    }

    private void ShowCritEffect(Node target) { }
}
```

## 4. 设计模式与最佳实践

### 4.1 伪 ECS 核心模式

#### Entity (实体) = Node + Component + Data

**传统 OOP 方式（❌ 不推荐）**:

```csharp
public partial class Enemy : CharacterBody2D
{
    private float _maxHp = 100;
    private float _currentHp = 100;
    private float _speed = 200;
    private Vector2 _velocity;

    public override void _PhysicsProcess(double delta)
    {
        // 移动逻辑
        // 战斗逻辑
        // AI 逻辑
        // ... 所有逻辑混在一起，难以复用
    }
}
```

**伪 ECS 方式（✅ 推荐）**:

```csharp
public partial class Enemy : CharacterBody2D
{
    // 组件引用
    private HealthComponent _health;
    private VelocityComponent _velocity;
    private FollowComponent _follow;
    private HurtboxComponent _hurtbox;

    // 动态数据
    private Data _data;

    // 静态配置
    [Export] public EnemyData EnemyData { get; set; }

    public override void _Ready()
    {
        // 获取组件
        _health = GetNode<HealthComponent>("HealthComponent");
        _velocity = GetNode<VelocityComponent>("VelocityComponent");
        _follow = GetNode<FollowComponent>("FollowComponent");
        _hurtbox = GetNode<HurtboxComponent>("Hurtbox");

        // 初始化动态数据
        _data = this.GetData();
        _data.Set("IsBoss", EnemyData.IsBoss);
        _data.Set("Reward", EnemyData.ExpReward);

        // 从配置初始化组件
        _health.MaxHp = EnemyData.MaxHp;
        _velocity.Speed = EnemyData.Speed;

        // 监听事件
        _health.Died += OnDied;
    }

    public override void _PhysicsProcess(double delta)
    {
        // 简洁的逻辑，委托给组件
        var direction = _follow.GetDirectionToTarget();
        _velocity.MoveToward(direction, (float)delta);
        Velocity = _velocity.Velocity;
        MoveAndSlide();
    }

    private void OnDied()
    {
        // ✅ 使用 C# Event 触发全局事件
        EventBus.TriggerEnemyDied(this, GlobalPosition);
        ObjectPoolManager.ReturnToPool(this);
    }
}
```

**优势**:

- ✅ 组件可复用（玩家和敌人共享 HealthComponent）
- ✅ 逻辑清晰，易于维护
- ✅ 便于测试（可单独测试组件）
- ✅ 支持可视化编辑（在编辑器中添加/移除组件）

### 4.2 组件通信模式

**核心原则**: **优先使用 C# Event，避免过度使用 Godot Signal**

#### 方式 1: C# Event - 推荐用于组件间通信（⭐⭐⭐⭐⭐）

```csharp
using System;

// HealthComponent 定义事件
public partial class HealthComponent : Node
{
    // ✅ 使用 C# Event
    public event Action? Died;
    public event Action<float>? Damaged;

    public void TakeDamage(float damage)
    {
        CurrentHp -= damage;
        Damaged?.Invoke(damage);  // 触发事件

        if (CurrentHp <= 0)
        {
            Died?.Invoke();
        }
    }
}

// Enemy 监听事件
public partial class Enemy : CharacterBody2D
{
    private HealthComponent _health;

    public override void _Ready()
    {
        _health = GetNode<HealthComponent>("HealthComponent");

        // ✅ 订阅事件
        _health.Died += OnDied;
        _health.Damaged += OnDamaged;
    }

    private void OnDied()
    {
        GD.Print("敌人死亡");
        EventBus.TriggerEnemyDied(this, GlobalPosition);
    }

    private void OnDamaged(float damage)
    {
        GD.Print($"受到伤害: {damage}");
    }

    // ⚠️ 重要：必须解绑，防止内存泄漏
    public override void _ExitTree()
    {
        if (_health != null)
        {
            _health.Died -= OnDied;
            _health.Damaged -= OnDamaged;
        }
    }
}
```

**优势**:

- ✅ 性能最好（编译期优化）
- ✅ 类型安全（编译期检查）
- ✅ 代码简洁
- ✅ 支持 Lambda 表达式

**注意事项**:

- ⚠️ 必须在 `_ExitTree()` 中解绑，否则内存泄漏
- ⚠️ 使用 `?.Invoke()` 防止空引用

### 4.3 数据管理模式

#### 静态配置 (Resource) vs 动态数据 (Data)

**静态配置 (Resource)** - 用于设计时确定的数据


**动态数据 (Data)** - 用于运行时变化的数据

```csharp
// 运行时修改
player.GetData().Set("CurrentDamage", 10);
player.GetData().Add("CurrentDamage", 5);  // Buff 加成
```

**组合使用**:

```csharp

        data.Set("Cooldown", WeaponData.Cooldown);

    }
```

### 4.5 性能优化要点

#### 1. 避免频繁的 GetNode 调用

**❌ 不好的做法**:

```csharp
public override void _Process(double delta)
{
    GetNode<HealthComponent>("Health").TakeDamage(1);  // 每帧查找
}
```

**✅ 好的做法**:

```csharp
private HealthComponent _health;

public override void _Ready()
{
    _health = GetNode<HealthComponent>("Health");  // 只查找一次
}

public override void _Process(double delta)
{
    _health.TakeDamage(1);
}
```

#### 2. 使用对象池

**必须使用对象池的场景**:

- 子弹（生命周期 < 5 秒）
- 敌人（频繁生成/销毁）
- 掉落物
- 粒子特效节点
- UI 伤害数字

#### 3. 批量处理

```csharp
// ❌ 逐个处理
foreach (var enemy in enemies)
{
    enemy.TakeDamage(10);
}

// ✅ 批量处理（如果可能）
DamageSystem.Instance.ApplyAreaDamage(enemies, 10, position, radius);
```

#### 4. 使用 Data 而非 Metadata

**❌ Godot 的 Metadata（性能较差）**:

```csharp
node.SetMeta("IsBoss", true);
var isBoss = (bool)node.GetMeta("IsBoss");
```

**✅ 使用 Data 容器（性能更好）**:

```csharp
node.GetData().Set("IsBoss", true);
var isBoss = node.GetData().Get<bool>("IsBoss");
```

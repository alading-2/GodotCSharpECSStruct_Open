# Godot 复刻项目 - 框架索引

## 📖 核心架构：Godot 伪 ECS (Pseudo-ECS)

本项目采用 **"Godot 伪 ECS"** 混合架构：利用 Godot 强大的 Scene Tree 处理渲染和物理（Entity），配合组件化（Component）和数据驱动（Data）实现逻辑解耦。

> **架构哲学**：保留 Godot 的易用性，引入 ECS 的解耦和数据驱动优势。

---

## 1. ECS 框架体系

### 1.1 Entity (实体)

**核心定义**：**"Scene 即 Entity"**。
在本项目中，Entity 不是一个基类，而是一个实现了 `IEntity` 接口的 Godot Scene（.tscn）。

**核心机制**：
- **EntityManager** - 实体的生命周期管家。负责从对象池生成 (`Spawn`)、注册绑定 (`Register`) 和回收销毁 (`Destroy`)。
- **EntityRelationshipManager** - 关系总管。维护 Entity-Component 和 Entity-Entity 的三索引结构，实现 O(1) 复杂查询。

### 1.2 Component (组件)

**核心定义**：**"逻辑的功能积木"**。
Component 是依附于 Entity 的功能单元，遵循 **"组合优于继承"**。组件应当是 **"无状态的"**（Stateless），只包含逻辑，状态存储于 Data 容器中。

**核心机制**：
- **纯逻辑设计** - 不持有私有状态（如 HP, Speed），所有数据从 `_entity.Data` 读写。
- **生命周期** - `OnComponentRegistered` (初始化) / `OnComponentUnregistered` (清理)。

**快速导航**：

### 1.3 Data (数据)

**核心定义**：**"动态状态容器"**。
Data 取代了传统的 C# 属性（Property），是一个挂载在 Entity 上的动态键值对容器。它解决了 ECS 中 "C" (Component Data) 的问题。

**核心价值**：
- **解耦逻辑与状态** - 任何组件都可以读写任何数据，无需互相引用。
- **自动化管理** - 支持从 Resource 批量加载，对象池回收时自动 `Clear`，天然支持 Buff/Debuff 系统。

**快速导航**：

- 💻 [Data.cs](../../Src/Tools/Data/Data.cs) - 数据容器实现
- 📘 [README.md](../../Src/Tools/Data/README.md) - 如何定义 Key 和读写数据

### 1.4 Event (事件)

**核心定义**：**"解耦通信总线"**。
基于 C# 原生事件封装的高性能、强类型事件系统，取代 Godot Signal 用于核心逻辑通信。

**核心机制**：
- **EventBus** - 支持优先级、一次性订阅 (`Once`) 和重入保护。
- **分层架构** - `GlobalEventBus` (全局跨系统) + `Entity.Events` (实体内部组件通信)。
- **零 GC 设计** - 使用 `readonly record struct` 传递事件数据，避免内存分配。

**快速导航**：

- 💻 [EventBus.cs](../../Src/ECS/Event/EventBus.cs) - 事件总线核心实现
-  [README_EventBus.md](../../Src/ECS/Event/README_EventBus.md) - 事件定义与订阅指南

### 1.5 System (系统)

**核心定义**：**"全局逻辑处理器"**。
处理跨实体或全局性的业务逻辑。

**核心模块**：
- **DamageSystem** - 职责链模式的伤害计算系统，处理 穿透 -> 暴击 -> 减伤 -> 实际伤害 的复杂流程。

**快速导航**：

## 2. 基础设施 (Infrastructure)

### ✅ AutoLoad (全局管理)
**定位**：上帝视角的单例容器。
管理所有 Manager 的初始化顺序和依赖注入，提供类型安全的 `Get<T>` 访问入口。
- 📋 [AutoLoad架构指南.md](./Tool/AutoLoad/AutoLoad%20架构指南.md)

### ✅ ObjectPool (对象池)
**定位**：高性能内存管理。
专为高频生灭对象（子弹、敌人、特显）设计，避免 GC 峰值，提升帧率。支持任意 Node 类型。
- 📋 [对象池.md](./Tool/对象池/对象池.md)
- 📘 [ObjectPool.md](../../Src/Tools/ObjectPool/ObjectPool.md)

### ✅ Tools (工具箱)
**定位**：通用开发套件。
- **Log** - 分级日志系统，Release 版本零开销。
- **Timer** - `TimerManager` 配置式高级计时器。
- **Input** - `InputManager` 统一输入映射封装。
- **Math** - 向量计算与数学扩展库。

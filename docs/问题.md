## 问题

### 1

- 属性系统
- DataMeta 很多时候不需要全部 Meta 都设置，是否要用?，可以设置为 Null
- DataType 不应该是枚举，需要设置为真正的类型，int,float,string,object,enum 等等，深度思考怎么做，哪些是游戏开发常用类型
- Data.Get `<float>`(DataKey.DPS)，还要自己传 `float`类型有点麻烦，这个是底层需要解决的问题，不是用户解决，DataMeta 已经定义了数据类型 Type，自动按照这个判断就行
- 角色添加装备，即 PlayerEntity 获取 ItemEntity，需要 EntityManager 建立关系，ItemEntity 本身是有属性 Data 的，这些数据应该通过 DataModifier 添加到 PlayerEntity 上，并且 DataModifier 要说明源头是 ItemEntity，DataModifier 现在明显没做到，但是修改器类型 ModifierTypeType 怎么自动判断呢？
- 有些 data 是从几个固定值选一个，DataMeta、DataType 等没有实现

### 2

- DataModifier 源头追踪实际是不重要，删掉，保持原来即可
- 枚举值约束支持问题理解错误，我的问题是有些 data 是从几个固定值选一个，实现可以用 list 或其他方式，枚举只是常量，实际是是数字

### 3

- 注册问题，
- 角色添加装备，即 PlayerEntity 获取 ItemEntity，需要 EntityManager 建立关系，ItemEntity 本身是有属性 Data 的，这些数据应该通过 DataModifier 添加到 PlayerEntity 上，并且 DataModifier 要说明源头是 ItemEntity，DataModifier 现在明显没做到，我觉得可以在 DataModifier 加一个属性类型是 object，值是添加修改器的源头类，你可以质疑，深度思考怎么做，
- Data.cs 中 RemoveModifierById 基本没用，所有用 ID 查询修改器的都没用，你想一下我怎么知道 ID 是什么，我只知道 EntityManager 添加关系时，将子 Entity 的 Data 改成修改器添加到 parentEntity，移除关系时移除之前增加的修改器。
- ApplyResource，EntityManager 有相似功能，EntityManager 需要更新 data 的内容，这个功能应该放在 Data 更好

### 4

- Data 是否需要扩展 Node，
- Data 如何应用到 Entity，之前是将扩展 Node

# 需求文档 - 组件系统 (Component System)

## 1. 简介

本文档定义了 Brotato 复刻项目的“伪 ECS”组件系统需求。组件系统是游戏框架的核心，通过提供可复用的功能模块，以组合而非继承的方式实现实体行为。系统必须支持 500+ 同屏实体的高性能场景，并与现有的 `Data` 容器、对象池和日志系统无缝集成。

## 2. 术语表

- **Component (组件)**：可复用的功能模块，继承自 Godot `Node`，通过组合方式附加到实体上。
- **Entity (实体)**：游戏中的基本单位（如玩家、敌人、子弹），由多个组件组合而成。
- **HealthComponent (生命值组件)**：管理实体的血量、伤害计算和死亡逻辑。
- **HitboxComponent (攻击判定组件)**：继承自 `Area2D`，定义攻击范围、伤害数值及击退属性。
- **HurtboxComponent (受击判定组件)**：继承自 `Area2D`，负责检测并响应来自 Hitbox 的攻击。
- **VelocityComponent (移动组件)**：管理实体的物理移动，包括速度、加速度和摩擦力。
- **AttributeComponent (属性组件)**：管理实体的基础属性（如伤害增益、速度加成）及属性修改器（Buff/Debuff）。
- **FollowComponent (跟随组件)**：实现 AI 实体跟随目标的逻辑。
- **PickupComponent (拾取组件)**：继承自 `Area2D`，实现物品的拾取检测和磁吸效果。
- **C# Event (C# 事件)**：采用 C# 原生事件机制进行组件间通信，而非 Godot 信号。
- **Data (动态数据)**：使用项目内置的动态数据容器存储运行时变化的属性。

## 3. 需求详情

### 需求 1: HealthComponent (生命值组件)

**用户故事：** 作为开发者，我需要一个可复用的生命值组件，以便轻松地为任何实体（玩家、敌人、可破坏物）添加血量管理。

#### 验收标准

1. **初始化**：当组件初始化时，`CurrentHp` 应设为 `MaxHp`。
2. **受到伤害**：调用 `TakeDamage` 传入正值时，减少 `CurrentHp` 并触发 `OnDamaged` 事件。
3. **死亡逻辑**：当 `CurrentHp` 归零或以下时，**仅触发一次** `OnDied` 事件。
4. **状态检查**：若 `CurrentHp` 已为 0，再次调用 `TakeDamage` 应被忽略。
5. **治疗逻辑**：调用 `Heal` 时增加 `CurrentHp`（不超过 `MaxHp`），并触发 `OnHealed` 事件。
6. **满血处理**：若当前已是满血，调用 `Heal` 应不产生任何变化。

### 需求 2: HitboxComponent (攻击判定组件)

**用户故事：** 作为开发者，我需要一个攻击判定组件来定义攻击区域，以便配置武器和子弹的伤害与击退。

#### 验收标准

1. **继承关系**：必须继承自 `Area2D` 以利用引擎的碰撞检测。
2. **导出属性**：通过 `[Export]` 暴露 `Damage`（伤害）、`Knockback`（击退力）和 `Source`（来源实体）。
3. **碰撞层级**：允许在编辑器中配置 `Collision Layer` 和 `Mask`。

### 需求 3: HurtboxComponent (受击判定组件)

**用户故事：** 作为开发者，我需要一个受击判定组件来检测敌方攻击。

#### 验收标准

1. **继承关系**：必须继承自 `Area2D`。
2. **自动伤害**：当 `HitboxComponent` 进入其区域时，自动调用关联 `HealthComponent` 的 `TakeDamage` 方法。
3. **无血量实体**：若实体没有血量组件但被击中，应触发 `OnHitReceived` 事件以便执行特殊逻辑（如特效），而不报错。
4. **无敌时间**：提供 `InvincibilityTime` 属性，防止在短时间内受到连续多次伤害。

### 需求 4: VelocityComponent (移动组件)

**用户故事：** 作为开发者，我需要一个处理移动物理的组件，以实现平滑的玩家和 AI 移动。

#### 验收标准

1. **加速移动**：调用 `MoveToward` 传入方向时，根据 `Acceleration` 向 `MaxSpeed` 加速。
2. **摩擦力**：调用 `ApplyFriction` 时，根据 `Friction` 减小当前速度。
3. **限速**：速度大小严禁超过 `MaxSpeed`。
4. **配置化**：通过导出属性配置最高速、加速度和摩擦力。

### 需求 5: AttributeComponent (属性组件)

**用户故事：** 作为开发者，我需要一个管理属性和修改器（Modifier）的组件，以实现复杂的 Buff 系统。

#### 验收标准

1. **基础属性**：存储伤害、速度、暴击率、暴击倍率的基础值。
2. **修改器支持**：支持 `AddModifier` 和 `RemoveModifier` 方法，并自动重新计算最终属性。
3. **计算逻辑**：支持“加法（Additive）”和“乘法（Multiplicative）”两种修改器类型。
4. **高性能**：属性获取方法应经过优化，避免每帧重复计算。

### 需求 6: FollowComponent (跟随组件)

**用户故事：** 作为开发者，我需要为 AI 实体提供跟随功能。

#### 验收标准

1. **目标追踪**：当设置有效的 `Target` (Node2D) 时，实时追踪其位置。
2. **方向计算**：`GetDirectionToTarget` 方法应返回指向目标的归一化向量。
3. **参数配置**：暴露 `FollowSpeed`（跟随速度）和 `StopDistance`（停止距离）。

### 需求 7: PickupComponent (拾取组件)

**用户故事：** 作为开发者，我需要一个拾取组件，以便实现经验值、金币等掉落物的收集。

#### 验收标准

1. **检测半径**：继承 `Area2D` 检测采集者的进入。
2. **磁吸效果**：支持将掉落物以 `MagnetSpeed` 向采集者方向平滑移动。
3. **拾取触发**：当采集者重叠时触发 `OnPickedUp` 事件。

### 需求 8: 组件集成与规范

**用户故事：** 我希望组件系统与现有框架保持一致。

#### 验收标准

1. **动态数据**：所有组件必须通过 `Node.GetData()` 扩展方法使用动态数据容器存储运行时状态。
2. **通信机制**：**强制使用 C# 原生事件**进行组件间通信，严禁使用 Godot 信号（Signal）以保证性能。
3. **内存安全**：当组件从场景树移除时，必须在 `_ExitTree` 中解绑所有事件，防止内存泄漏。

# Godot 碰撞层级说明（本项目）

## 一、Layer 和 Mask 是什么？

> **Layer = 我是谁，Mask = 我关注谁**

- **`collision_layer`**：这个物体"挂"在哪个层，代表它的身份标签。
- **`collision_mask`**：这个物体"扫描"哪些层，只有扫描到的层才会触发碰撞事件。

**碰撞发生的条件**：A 的 `mask` 包含 B 的 `layer`，**或者** B 的 `mask` 包含 A 的 `layer`，二者满足其一即可触发。

---

## 二、本项目的 8 个层定义

```text
Layer 1 (值=1)    → Terrain          地形/障碍物
Layer 2 (值=2)    → Player           玩家物理体
Layer 3 (值=4)    → Enemy            敌人物理体
Layer 4 (值=8)    → PlayerHurtbox    玩家受伤感应区（HurtComponent 挂玩家时）
Layer 5 (值=16)   → PlayerPickup     玩家拾取感应区身份标签
Layer 6 (值=32)   → Projectile       子弹/投射物物理体
Layer 7 (值=64)   → EnemyHurtbox     敌人受伤感应区（HurtComponent 挂敌人时）
Layer 8 (值=128)  → WeaponHitbox     武器/技能命中感应区（HitComponent）
```

> 层的数值是 **2 的幂次**（位掩码），多层叠加直接相加。例如 `mask=36` = `4+32` = Enemy + Projectile。

所有常量定义见 `Data/DataKey/Base/CollisionLayers.cs`，**代码中禁止直接写魔法数字**。

---

## 三、通用化设计原则

> **场景文件（.tscn）中碰撞层一律清零，由代码在运行时根据 Entity 阵营自动配置。**

### 为什么这样做？

同一个 `HurtComponent.tscn` 可以挂在玩家、敌人、召唤物上，无需为每种实体单独设置场景文件。
注册时 `OnComponentRegistered` 读取 `DataKey.Team`，调用 `ConfigureCollisionByTeam()` 一次性完成配置：

```csharp
// HurtComponent 内部自动配置（无需外部干预）
case Team.Player: Layer=PlayerHurtbox(8),  Mask=Enemy(4)|Projectile(32)
case Team.Enemy:  Layer=EnemyHurtbox(64),  Mask=Player(2)|WeaponHitbox(128)
```

### 碰撞关闭时机

| 时机 | 操作 | 原因 |
| ---- | ---- | ---- |
| Entity 死亡 | `Monitoring = false` | 防止死亡后继续触发接触伤害 |
| 对象池归还（`OnPoolRelease`） | `ProcessMode = Disabled`（ObjectPool 自动处理） | 完全停止节点，无需额外操作 |
| 无敌帧 | `Monitoring = false` | 暂停检测，不影响层级身份 |
| 子弹命中后 | `Monitoring = false` | 防止同一子弹多次命中 |

> **区分**：`Monitoring=false` 停止主动检测（自己不感知别人），`Monitorable=false` 停止被动感知（别人感知不到自己），`ProcessMode=Disabled` 完全停止（对象池归还专用）。

### 2026-03 碰撞架构（当前）

#### 碰撞类型注册表（CollisionTypeRegistry）

所有碰撞预设统一存放于 `Data/Data/Collision/`，由 `ResourceGenerator` 自动扫描生成：
- **枚举**：`CollisionType`（Custom=0，其余按文件名字母序编号）
- **注册表**：`CollisionTypeRegistry`
  - `FromLayer(uint layer)` → `CollisionType`（O(1)，layer=0 返回 Custom）
  - `GetLayerMask(CollisionType)` → `(Layer, Mask)`（O(1)）

```text
Data/Data/Collision/
  Unit/
    EnemyCollision.tscn      → CharacterBody2D, layer=4(Enemy), mask=5(Terrain+Enemy)，无内嵌形状
    PlayerCollision.tscn     → CharacterBody2D, layer=2(Player), mask=1(Terrain)，无内嵌形状
  Effect/
    EffectCollision.tscn     → Area2D, layer=0, mask=0（mask 由业务代码配置），无内嵌形状
  Sensor/
    PlayerHurtboxSensor.tscn → Area2D, layer=8(PlayerHurtbox), mask=4(Enemy)
    EnemyHurtboxSensor.tscn  → Area2D, layer=64(EnemyHurtbox), mask=128(WeaponHitbox)
    PlayerPickupSensor.tscn  → Area2D, layer=16(PlayerPickup), mask=0
```

SpriteFramesGenerator 将碰撞模板实例化到生成的动画场景时，会自动在实例下添加 `CollisionShape2D`（默认 CapsuleShape2D），结构如下：

```text
AnimatedSprite2D "豺狼人"
  └─ CollisionShape2D      ← 碰撞模板实例（CharacterBody2D/Area2D）
       └─ CollisionShape2D ← 由生成器动态注入（默认 CapsuleShape2D，各角色独立调整）
```

智能更新时自动保留手动调整的 shape / transform / disabled，重新生成不会覆盖。

#### 实体节点树结构

```text
Entity Root（CharacterBody2D）
  ├─ CollisionShape2D          ← 直接内嵌，shape 在实体场景中硬编码
  ├─ Component/
  │   ├─ CollisionSensorComponent（Area2D）
  │   │     [Export] SensorType = PlayerHurtboxSensor
  │   │     → _Ready 自动应用 layer=8, mask=4
  │   │     → OnNodeEntered 发布 CollisionEntered(SensorType=PlayerHurtboxSensor)
  │   ├─ CollisionComponent（Node）← 桥接 VisualRoot 内碰撞信号
  │   │     → OnComponentRegistered 找 VisualRoot/"CollisionShape2D"
  │   │     → FromLayer(area.CollisionLayer) 派生 CollisionType（一次缓存）
  │   │     → OnBodyEntered 发布 CollisionEntered(CollisionType=派生值)
  │   └─ ContactDamageComponent / PickupComponent（可选）
  └─ VisualRoot（SpriteFramesGenerator 产物：AnimatedSprite2D）
        └─ CollisionShape2D（节点名称，固定）
              ← 实例化自 Data/Data/Collision/ 对应 .tscn
              ← layer/mask 已在场景中预配置，无 shape（插槽）
```

#### CollisionSensorComponent 使用方式

- **Inspector 选择 SensorType**：从 `CollisionType` 枚举中选择预设类型（如 `PlayerHurtboxSensor`）
- **_Ready 自动应用**：`GetLayerMask(SensorType)` → 设置 `CollisionLayer/CollisionMask`
- **Custom 模式**：SensorType=Custom 时不修改 layer/mask，使用场景手动配置值
- **预设场景**：`Data/Data/Collision/Sensor/*.tscn` 可直接拖入 Component 节点

#### 事件数据（CollisionEnteredEventData）

```csharp
Source         → 事件源实体（拥有 CollisionSensorComponent/CollisionComponent 的 Entity）
Target         → 碰撞目标节点（物理节点，可能是 IEntity 也可能不是）
CollisionType  → 碰撞来源类型（由发布者写入，消费者据此过滤）
```

消费者过滤示例：
- `ContactDamageComponent` → 处理 `PlayerHurtboxSensor`、`EnemyHurtboxSensor`（当前暂不过滤，待 PlayerEntity.tscn 设置 SensorType 后开启）
- `DestroyOnCollision`（EntityMovementComponent）→ 订阅 `CollisionEntered`（Area2D）或检测 `MoveAndSlide`（CharacterBody2D），触发 `MovementCollision` 事件后回收实体

## 四、各对象的 Layer / Mask 设置一览

| 对象 | 节点/组件 | collision_layer | collision_mask | 配置方式 |
| ---- | ---- | -------------- | -------------- | ---- |
| **PlayerEntity** | `CharacterBody2D` | `2` (Player) | `1` (Terrain) | 实体场景直接配置 |
| **PlayerEntity** | `CollisionSensorComponent`（Area2D） | `8` (PlayerHurtbox) | `4` (Enemy) | 实体场景直接配置 |
| **PlayerEntity** | `PickupComponent`（Area2D） | `16` (PlayerPickup) | `0` | 场景文件固定 |
| **EnemyEntity** | `CharacterBody2D` | `4` (Enemy) | `5` (Terrain+Enemy) | 实体场景直接配置 |
| **掉落物/经验球** | `Area2D` | `0` | `16` (PlayerPickup) | 场景文件固定 |
| **子弹** | `Area2D`/`CharacterBody2D` | `32` (Projectile) | `64` (EnemyHurtbox) | 场景文件固定 |
| **武器命中区** | `HitComponent`（Area2D） | `128` (WeaponHitbox) | `64` (EnemyHurtbox) | **代码自动配置**（待实现） |

## 五、核心碰撞链路

### 链路 1：近战攻击打敌人（代码距离判定）

```text
AI 行为树 → AttackComponent
  └─ 距离计算：body.GlobalPosition.DistanceTo(target.GlobalPosition)
  └─ 命中后 → DamageService.Process(damageInfo)
```

**关键设计**：近战伤害**不依赖碰撞层**，由 `AttackComponent` 的距离校验 + `DamageService` 管道处理。

---

### 链路 2：接触伤害（通用，双向）

```text
任意单位 CharacterBody2D
  └─ Layer = Player(2) 或 Enemy(4)

CollisionSensorComponent（Area2D）
  └─ layer/mask/shape 在实体场景中直接配置（不依赖 VisualRoot）
  └─ body_entered → 发出 CollisionEntered(Target, TargetEntity)

ContactDamageComponent
  └─ 读取 TargetEntity；缺失时再动态回溯宿主 Entity
  └─ TimerManager.Loop(0.5s) → DamageService.Process() 造成接触伤害
  └─ 自身死亡时 → Monitoring=false，停止检测
```

**关键设计**：
- `CollisionSensorComponent` 只负责碰撞采样与事件分发，`ContactDamageComponent` 负责接触伤害节流与结算
- 碰撞事件同时保留原始 `Target` 与解析后的 `TargetEntity`，避免子碰撞节点破坏既有业务逻辑
- 伤害值优先读取 `DataKey.ContactDamage`，回退到 `DataKey.FinalAttack`
- `IsHostile()` 判断对立阵营，中立单位不触发伤害

---

### 链路 3：玩家拾取掉落物

```text
玩家 PickupComponent（Area2D）
  └─ Layer = 16 (PlayerPickup)  ← 作为身份标签

掉落物 Area2D
  └─ Mask = 16 (PlayerPickup)  ← 主动扫描玩家拾取区
  └─ area_entered → 触发磁吸/拾取逻辑
```

---

### 链路 4：子弹/武器命中敌人（待实现）

```text
子弹/武器 HitComponent（Area2D）
  └─ Layer = 128 (WeaponHitbox)
  └─ Mask  = 64  (EnemyHurtbox)  ← 扫描敌人受伤区

敌人 HurtComponent（Area2D）
  └─ Layer = 64 (EnemyHurtbox)
  └─ area_entered → DamageService.Process()
  └─ 命中后 → Monitoring=false（防止多次命中）
```

---

### 链路 5：敌人目标感知（TargetSelector，无碰撞层）

```text
AI 行为树 → FindNearestPlayer()
  └─ TargetSelector.Query(Circle, DetectionRange, TeamFilter.Enemy)
  └─ 遍历 EntityManager 注册的所有实体，距离+阵营过滤
  └─ 结果缓存到 DataKey.TargetNode
```

**关键设计**：敌人感知玩家**完全不依赖碰撞层**，由 `TargetSelector` 纯代码实现。

## 六、为什么 Enemy 保持 CharacterBody2D？

| 需求 | CharacterBody2D | Area2D |
| ---- | --------------- | ------ |
| 被地形阻挡（不穿墙） | ✅ 天然支持 | ❌ 需手写 |
| 敌人之间互相推开 | ✅ move_and_slide | ❌ 穿模 |
| 感知玩家 | 不需要（TargetSelector 负责） | — |
| 伤害判定 | 不需要（HurtComponent 负责） | — |

---

## 七、层级设计总结

```text
Layer 1  (1)    Terrain          ← 地图边界/障碍物（阻挡物理体）
Layer 2  (2)    Player           ← 玩家物理体（在地形上行走）
Layer 3  (4)    Enemy            ← 敌人物理体（地形阻挡+互相推开）
Layer 4  (8)    PlayerHurtbox    ← 玩家受伤感应区（HurtComponent@Player）
Layer 5  (16)   PlayerPickup     ← 玩家拾取感应区身份标签（PickupComponent）
Layer 6  (32)   Projectile       ← 子弹/投射物物理体
Layer 7  (64)   EnemyHurtbox     ← 敌人受伤感应区（HurtComponent@Enemy）
Layer 8  (128)  WeaponHitbox     ← 武器/技能命中感应区（HitComponent）
```

**设计原则**：

- 物理移动（CharacterBody2D）和伤害判定（Area2D）完全分离
- 近战伤害、目标感知均由代码系统负责，不走碰撞层
- **感应区组件（HurtComponent/HitComponent）场景文件层级清零，运行时由 Team 自动配置**
- 碰撞层常量统一定义在 `CollisionLayers.cs`，禁止魔法数字

## 八、快速设置参考

代码中统一使用 `CollisionLayers` 常量，禁止直接写数字：

```csharp
// 推荐写法
CollisionLayer = CollisionLayers.PlayerHurtbox;
CollisionMask  = CollisionLayers.PlayerHurtboxMask; // = Enemy | Projectile

// 组合写法
CollisionMask = CollisionLayers.Player | CollisionLayers.WeaponHitbox;

// 关闭检测（无敌帧/死亡）
Monitoring = false;

// 关闭被感知（隐身/穿透）
Monitorable = false;
```

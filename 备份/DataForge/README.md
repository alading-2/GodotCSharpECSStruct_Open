# DataForge Enhanced - 数据锻造台

> **专业级 Godot C# 数据编辑工具** - Excel级别的编辑体验，自动生成类型安全的 C# 代码。

## 🎉 v2.0 增强版特性

- ✨ **Excel风格Sheet标签页** - 底部标签快速切换，告别下拉框
- 🖱️ **完整右键上下文菜单** - 插入、删除、复制、粘贴，操作就近
- ⌨️ **专业快捷键系统** - Ctrl+S/Z/Y/C/V/X/D/N/G，效率翻倍
- ↶ **撤销/重做机制** - 最多50步历史，大胆操作不怕错
- 📊 **多行选择与批量操作** - 像Excel一样选择多行
- 🎨 **智能视觉增强** - 类型着色、图标标识、空值提示
- 🔍 **全字段搜索** - 不仅搜索ID，任意字段都能搜

## 🚀 快速开始

### 1. 启用插件

1. 打开 Godot 编辑器
2. 点击菜单 `Project` → `Project Settings` → `Plugins`
3. 找到 `DataForge` 插件，勾选 `Enable`

### 2. 打开工具

1. 点击菜单 `Project` → `Tools` → `DataForge - 数据锻造台`
2. 或者等待编译完成后，工具会自动出现在 Tools 菜单

### 3. 使用流程

```
┌─────────────────────────────────────────────────────────────┐
│  1. 选择数据类型 (敌人/玩家/技能)                             │
│  2. 点击 "➕ 新增行" 添加新配置                               │
│  3. 直接在表格中编辑数据                                      │
│  4. 点击 "💾 保存存档" 保存到 JSON                            │
│  5. 点击 "🔥 生成代码" 生成 C# 文件                           │
└─────────────────────────────────────────────────────────────┘
```

## 🎯 快速上手

### 基本操作

1. **切换数据表**: 点击底部Sheet标签（👹敌人、🎮玩家、⚡技能）
2. **搜索数据**: 在顶部搜索框输入关键词，实时过滤
3. **新增行**: 点击 `➕ 新增` 或按 `Ctrl+N`
4. **编辑单元格**: 直接点击单元格开始编辑
5. **删除行**: 选中行后按 `Delete` 或右键选择删除
6. **保存**: 按 `Ctrl+S` 保存到JSON
7. **生成代码**: 按 `Ctrl+G` 生成C#代码

### 快捷键速查

| 快捷键 | 功能 |
|--------|------|
| `Ctrl+S` | 保存到JSON |
| `Ctrl+G` | 生成C#代码 |
| `Ctrl+Z` | 撤销 |
| `Ctrl+Y` | 重做 |
| `Ctrl+N` | 新增行 |
| `Ctrl+D` | 复制行 |
| `Ctrl+C/V/X` | 复制/粘贴/剪切 |
| `Delete` | 删除选中行 |

### 右键菜单

在任意行上右键可以：
- ➕ 在上方/下方插入行
- 📋 复制/剪切/粘贴行
- 🗑️ 删除选中行
- 🧹 清除单元格内容
- 📊 查看行详情

## 💡 使用技巧

### 枚举字段输入

直接输入枚举值名称（不含类型名）：
```
Team 列: Enemy (不是 Team.Enemy)
AbilityType 列: Passive (不是 AbilityType.Passive)
```

### SpawnRule 字段输入

支持两种格式：

**简化格式**:
```
MinWave:1,MaxWave:-1,Interval:2.0,Count:3,Strategy:Circle
```

**JSON 格式**:
```json
{"MinWave":1,"MaxWave":-1,"SpawnInterval":2.0,"SingleSpawnCount":3,"Strategy":"Circle"}
```

### VisualScenePath 字段

只需输入资源名称，代码生成时会自动包装：
```
输入: 鱼人
生成: ResourceManagement.GetPath("鱼人")
```

---

## ⚠️ 注意事项

1. **每次生成会覆盖目标文件** - 不要手动修改生成的 .cs 文件
2. **JSON 是数据源** - 建议将 `Data/Source/*.json` 加入版本控制
3. **生成后需要重新编译** - Godot 会自动扫描并编译
4. **枚举值必须正确** - 错误的枚举值会导致编译错误

## 🔄 扩展模板

如需添加新的数据类型，编辑 `DataForgeWindow.cs` 中的 `Templates` 数组：

```csharp
new DataForgeTemplate
{
    Name = "NewData",
    DisplayName = "新数据类型",
    SavePath = "res://Data/Source/NewData.json",
    OutputPath = "res://Data/Data/NewData.cs",
    ClassName = "NewData",
    FieldOverrides = new Dictionary<string, FieldOverride>
    {
        // 配置特殊字段...
    }
}
```

## 🎯 设计理念

DataForge Enhanced 的目标不是复制 Excel，而是为**游戏配置编辑**量身定制：

1. **专注核心价值**: 只实现对游戏配置有用的功能
2. **深度集成**: 与Godot编辑器无缝配合
3. **类型安全**: 自动生成强类型C#代码
4. **高效工作流**: 快捷键、批量操作、撤销重做
5. **容错设计**: 验证、提示、撤销机制

**核心优势**: 纯Godot UI实现，无需外部依赖，性能优秀，与项目深度集成！
